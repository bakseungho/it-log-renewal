## 🔄 3. 사용자 플로우 설계

### 3.1 주요 사용자 시나리오

#### 시나리오 1: 솔루션 탐색 → 상세 정보 → 문의
```
[진입점] 메인 페이지
   ↓
[1단계] 히어로 섹션
   ├─ "솔루션 보기" 클릭 → 솔루션 섹션으로 스크롤
   └─ "문의하기" 클릭 → support/contact.html (빠른 전환)
   ↓
[2단계] 원스탑 솔루션 섹션
   └─ "전체 솔루션 보기" → solutions/one-stop-solution.html
   ↓
[3단계] 원스탑 솔루션 페이지
   ├─ 5개 솔루션 개요 확인
   └─ 관심 솔루션 클릭 → 개별 솔루션 페이지
   ↓
[4단계] 개별 솔루션 페이지 (예: AI 영상 관제)
   ├─ 시스템 개요 및 주요기능 확인
   ├─ 구성품 및 구성도 검토
   └─ 하단 CTA "문의하기" 클릭
   ↓
[전환] support/contact.html
   └─ 문의 양식 작성 또는 메일 발송
```

### 4.2 솔루션 페이지 정보 제공 순서

#### 표준 구조 (모든 솔루션 페이지 공통)
```
1. 페이지 헤더 (Breadcrumb + 타이틀)
   └─ 사용자 위치 인지 및 네비게이션 지원

2. 시스템 개요 (What)
   ├─ 꾸미기 문구 (감성적 접근)
   ├─ 설명 텍스트 (문제 정의 + 솔루션 제시)
   └─ 대표 이미지 (시각적 이해)
   
3. 시스템 주요기능 (How)
   └─ 4개 기능 카드 (구체적 기능 설명)
   
4. 시스템 구성도 (Architecture)
   └─ 다이어그램 (기술적 이해)
   
5. 시스템 구성품 (Components)
   └─ 제품 카드 그리드 (실제 제품 확인)
   
6. 주요 시공사례 (Proof)
   └─ 관련 프로젝트 3개 (신뢰 구축)
   
7. 문의 유도 CTA (Action)
   └─ 전환 유도 (다음 단계 안내)
```

#### 정보 제공 원칙
1. **역피라미드 구조**: 중요한 정보를 먼저 제공
2. **스캔 가능성**: 제목, 아이콘, 이미지로 빠른 이해 지원
3. **점진적 상세화**: 개요 → 기능 → 구성 → 사례 순으로 깊이 증가
4. **명확한 CTA**: 각 섹션 후 다음 행동 유도

### 4.3 CTA 배치 전략

#### CTA 유형별 우선순위
```
Primary CTA (주요 전환)
├─ "문의하기" / "무료 상담 신청"
├─ 색상: Primary Blue (#0066FF)
└─ 위치: 히어로, 솔루션 페이지 하단, 고객지원 섹션

Secondary CTA (보조 행동)
├─ "솔루션 보기" / "자세히 보기"
├─ 색상: Outline 또는 Secondary
└─ 위치: 히어로, 섹션 하단

Tertiary CTA (추가 정보)
├─ "더보기" / "전체 보기"
├─ 색상: Text Link
└─ 위치: 시공사례, 고객사 섹션
```

#### 페이지별 CTA 배치
```
메인 페이지
├─ 히어로 슬라이드 1: "솔루션 보기" (Secondary) + "문의하기" (Primary)
├─ 히어로 슬라이드 2: "자세히 보기" (Secondary)
├─ 히어로 슬라이드 3: "환경센서 보기" (Primary) + "출입통제 보기" (Outline)
├─ 히어로 슬라이드 4: "원격지원" (Secondary) + "문의하기" (Primary)
├─ 원스탑 솔루션 섹션: "전체 솔루션 보기" (Secondary)
├─ 시공사례 섹션: "더보기" (Tertiary)
└─ 고객지원 섹션: "원격지원" (Secondary) + "문의하기" (Primary)

솔루션 페이지
├─ 페이지 상단: Breadcrumb (네비게이션)
├─ 시스템 개요: "견적 요청하기" (선택적)
└─ 페이지 하단: "문의하기" (Primary) - 공통 CTA 영역

시공사례 페이지
├─ 필터링 탭: 카테고리별 탐색
└─ 프로젝트 카드: 클릭 시 상세 정보 또는 관련 솔루션

회사소개 페이지
└─ 페이지 하단: "문의하기" (Primary) - 공통 CTA 영역
```

## 📈 7. 예상 효과 및 KPI

### 7.1 사용자 경험 개선

#### 정보 접근성 향상
- **현재**: 일부 메뉴가 비활성화되어 정보 접근 불가
- **개선 후**: 모든 메뉴 활성화로 완전한 정보 제공
- **예상 효과**: 이탈률 15~20% 감소

#### 탐색 효율성 증가
- **현재**: 시공사례가 메인 페이지 섹션으로만 존재
- **개선 후**: 독립 페이지 + 필터링 기능으로 상세 탐색 가능
- **예상 효과**: 페이지 체류 시간 30% 증가

#### 전환 경로 최적화
- **현재**: 히어로 → 솔루션 카드 → 상세 페이지 (2단계)
- **개선 후**: 히어로 → 원스탑 솔루션 → 개별 솔루션 (3단계, 더 명확)
- **예상 효과**: 문의 전환율 10~15% 증가

### 7.2 비즈니스 목표 달성

#### 리드 생성 증가
```
개선 요소:
- 명확한 CTA 배치 (각 페이지 하단)
- 원스탑 솔루션 페이지로 통합 가치 제안 강화
- 시공사례 페이지로 신뢰 구축

예상 결과:
- 월간 문의 건수 20~30% 증가
- 견적 요청 건수 15~25% 증가
```

#### 브랜드 신뢰도 향상
```
개선 요소:
- 안전보건 경영방침 페이지 추가 (전문성 강화)
- 시공사례 페이지 확장 (실적 증명)
- 체계적인 정보 구조 (전문 기업 이미지)

예상 결과:
- 공공기관/대기업 입찰 경쟁력 강화
- 브랜드 검색량 10~15% 증가
```

#### SEO 성과 개선
```
개선 요소:
- 4개 신규 페이지 추가 (인덱싱 증가)
- Breadcrumb 구조화 데이터 (검색 노출 개선)
- 내부 링크 구조 강화 (크롤링 효율 증가)

예상 결과:
- 자연 검색 트래픽 25~35% 증가
- 주요 키워드 순위 상승
```

### 7.3 측정 가능한 KPI

#### 정량적 지표
```
1. 트래픽 지표
   - 페이지뷰: 30% 증가 목표
   - 세션 시간: 40% 증가 목표
   - 이탈률: 20% 감소 목표

2. 전환 지표
   - 문의 전환율: 2.5% → 3.5% (40% 증가)
   - 원격지원 클릭률: 1.5% → 2.5% (67% 증가)
   - 솔루션 페이지 도달률: 50% → 70%

3. 참여 지표
   - 평균 페이지 깊이: 3페이지 → 5페이지
   - 시공사례 페이지 체류 시간: 신규 측정
   - 솔루션 페이지 간 이동률: 20% → 40%
```

#### 정성적 지표
```
1. 사용자 만족도
   - 정보 찾기 용이성 (설문조사)
   - 네비게이션 명확성 (사용성 테스트)
   - 전반적 만족도 (NPS 점수)

2. 비즈니스 피드백
   - 영업팀 리드 품질 평가
   - 고객 문의 내용 구체성
   - 입찰 제안서 작성 효율성
```
