## 1. 와이어프레임 개요

### 1.1 목적

- 페이지 구조 및 레이아웃 정의
- 콘텐츠 배치 및 우선순위 결정
- 사용자 플로우 검증
- 개발팀과의 커뮤니케이션 도구

### 1.2 표기 규칙

```
[LOGO]          : 로고 이미지
[IMAGE]         : 이미지 영역
[ICON]          : 아이콘
[CTA Button]    : 행동 유도 버튼
[Link]          : 텍스트 링크
---             : 구분선
[ ]             : 입력 필드
[v]             : 드롭다운
```

## 6. 반응형 고려사항

### 6.1 모바일 레이아웃 변화

**데스크톱 → 모바일**
- 4열 그리드 → 1열 스택
- 좌우 배치 → 상하 배치
- 호버 효과 → 터치 최적화
- 드롭다운 메뉴 → 햄버거 메뉴

### 6.2 터치 타겟 크기

- 최소 44x44px
- 버튼 간 충분한 간격 (최소 8px)
- 스와이프 제스처 지원

## 7. 인터랙션 노트

### 7.1 스크롤 애니메이션

- 요소가 뷰포트에 진입 시 페이드인 + 슬라이드업
- 숫자 섹션: 카운트업 애니메이션
- 한 번만 실행 (재스크롤 시 반복 안 함)

### 7.2 호버 효과

- 카드: 상승 + 그림자 증가
- 버튼: 배경색 변화 + 약간 확대
- 링크: 밑줄 애니메이션

### 7.3 로딩 상태

- 이미지: 레이지 로딩 + 플레이스홀더
- 폼 제출: 로딩 스피너 표시
- 페이지 전환: 부드러운 페이드
