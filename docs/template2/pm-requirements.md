## 1. 비즈니스 요구사항

### 1.1 프로젝트 목적
- 스마트 건설 안전 솔루션 선도 기업으로서의 브랜드 이미지 강화
- B2B 고객의 의사결정을 지원하는 명확한 정보 제공
- 온라인을 통한 리드 생성 및 문의 전환율 향상
- 모바일 환경에서의 접근성 개선

### 1.2 비즈니스 목표
1. 문의 전환율 35% 향상
2. 월 평균 방문자 수 25% 증가
3. 평균 세션 시간 3분 30초 이상
4. 모바일 트래픽 55% 이상

### 1.3 성공 지표 (KPI)
- 월 평균 문의 건수
- 페이지뷰 및 세션 시간
- 이탈률 (35% 이하 목표)
- 재방문율 (35% 이상 목표)
- 주요 키워드 검색 순위

## 2. 사용자 요구사항

### 2.1 타겟 사용자

#### 페르소나 1: 구매 의사결정자
- **니즈**: 회사 신뢰도, 실적, 안전성 확인
- **요구사항**:
  - 시공사례 및 고객사 정보 명확히 제시
  - 인증 및 특허 정보 접근 용이
  - 회사 연혁 및 규모 확인 가능
  - 간편한 문의 경로

#### 페르소나 2: 현장 담당자
- **니즈**: 시스템 구성도, 기술 사양, 설치 사례
- **요구사항**:
  - 솔루션별 상세 정보 제공
  - 시스템 구성도 시각적 제공
  - 모바일에서 빠른 정보 확인
  - 원격지원 서비스 접근 용이

#### 페르소나 3: 구매/조달 담당자
- **니즈**: 빠른 문의, 명확한 연락처
- **요구사항**:
  - 문의 폼 간편하게 작성 가능
  - 연락처 정보 명확히 표시
  - 회사 기본 정보 쉽게 확인

### 2.2 사용자 시나리오

**시나리오 1: 솔루션 탐색**
1. 검색 엔진을 통해 메인 페이지 진입
2. 히어로 섹션에서 회사 슬로건 확인
3. 주요 솔루션 섹션에서 관심 솔루션 클릭
4. 솔루션 상세 페이지에서 시스템 구성도 및 사례 확인
5. 문의하기 버튼 클릭하여 문의 폼 작성

**시나리오 2: 회사 정보 확인**
1. 메인 페이지 진입
2. 헤더 메뉴에서 "회사소개" 클릭
3. 회사 개요, 연혁, 인증 정보 확인
4. 신뢰도 확인 후 문의하기

**시나리오 3: 원격지원 (기존 고객)**
1. 메인 페이지 또는 검색을 통해 진입
2. 헤더 또는 메인 페이지에서 "원격지원" 클릭
3. 원격지원 페이지에서 서비스 설명 확인
4. 원격지원 시작하기 버튼 클릭하여 외부 서비스 연결

## 3. 기능 요구사항

### 3.1 필수 기능 (P0)

#### F-001: 반응형 네비게이션
- 데스크톱: 드롭다운 메뉴
- 모바일: 햄버거 메뉴 (전체 화면 오버레이)
- 현재 페이지 하이라이트
- 스크롤 시 헤더 고정

#### F-002: 히어로 섹션 슬라이더
- 4개 슬라이드 자동 재생 (5초 간격)
- 좌우 화살표 및 인디케이터
- 각 슬라이드별 CTA 버튼
- 모바일 스와이프 제스처 지원

#### F-003: 솔루션 카드
- 4대 핵심 솔루션 카드 표시
- 호버 효과 (확대 또는 그림자)
- 클릭 시 해당 솔루션 페이지로 이동
- 스크롤 애니메이션

#### F-004: 숫자로 보는 신뢰 섹션
- 4개 데이터 항목 표시
- 스크롤 시 카운트업 애니메이션
- 반응형 그리드 배치

#### F-005: 시공사례 표시
- 메인 페이지: 대표 사례 3-4개
- 솔루션 페이지: 해당 솔루션 사례 3-4개
- 이미지 클릭 시 확대 보기

#### F-006: 문의 폼
- 입력 필드: 이름, 회사명, 연락처, 이메일, 제목, 내용
- 유효성 검사
- 개인정보 수집 동의
- 메일 발송 기능

#### F-007: 카카오맵 연동
- 오시는 길 페이지에 지도 표시
- 회사 위치 마커
- 길찾기 기능

#### F-008: 원격지원 외부 연결
- 원격지원 페이지에서 외부 서비스로 연결
- 새 창으로 열기


### 3.2 중요 기능 (P1)

#### F-009: 브레드크럼
- 현재 페이지 위치 표시
- 각 단계 클릭 시 해당 페이지로 이동

#### F-010: 고객사 섹션
- 주요 고객사 CI 로고 그리드 표시
- 호버 효과 (grayscale → 컬러)

#### F-011: 회사연혁 타임라인
- 2010년~2025년 연혁 시각화
- 스크롤 애니메이션
- 좌우 지그재그 배치 (데스크톱)

#### F-012: 스크롤 애니메이션
- 페이지 스크롤 시 요소 순차적 나타남
- 페이드인, 슬라이드업 효과

#### F-013: 상단으로 이동 버튼
- 스크롤 일정 거리 이상 시 버튼 표시
- 클릭 시 페이지 상단으로 스무스 스크롤

### 3.3 선택 기능 (P2)
- 다국어 지원
- 다크 모드
- 소셜 미디어 공유 버튼

## 4. 콘텐츠 요구사항

### 4.1 메인 페이지

#### 히어로 섹션
- 슬라이드 1: 브랜드 가치 (슬로건, 서브타이틀, 문구, CTA 2개)
- 슬라이드 2: 핵심 기술 (AI/통합안전, CTA 1개)
- 슬라이드 3: 현장 맞춤 솔루션 (환경센서, CTA 1개)
- 슬라이드 4: 신뢰와 유지보수 (CS, CTA 1개)
- 각 슬라이드별 배경 이미지

#### 주요 솔루션 섹션
- 4개 솔루션 카드 (아이콘, 제목, 설명, 링크)

#### 숫자로 보는 신뢰 섹션
- 설치 횟수, 연구 기간, 고객사 수, 기타 데이터

#### 고객사 섹션
- 주요 고객사 CI 로고 이미지

#### 시공사례 섹션
- 대표 프로젝트 3-4개 (이미지, 프로젝트명, 발주처, 설명)

### 4.2 솔루션 페이지 (6개)

각 솔루션 페이지별:
- 시스템 개요 (2-3 문단)
- 주요 특징 (3-5개 불릿 포인트)
- 적용 분야
- 시스템 구성도 이미지
- 하위 시스템 정보 (해당 시)
- 시공사례 3-4개

### 4.3 회사소개 페이지 (5개)

#### 회사소개
- 기업정보, 기업 이념, 안전보건 경영방침/목표
- 사업분야
- 고객사 CI 로고

#### CEO 인사말
- CEO 사진
- 인사말 전문 (3-5 문단)
- CEO 서명 이미지

#### 회사연혁
- 2010년~2025년 연도별 주요 성과

#### 인증 및 특허
- 인증서 이미지 및 명칭
- 특허 이미지 및 명칭

#### 오시는 길
- 주소 (도로명/지번)
- 연락처 (전화, 팩스, 이메일)
- 대중교통 이용 안내
- 카카오맵

### 4.4 고객지원 페이지 (2개)

#### 원격지원
- 서비스 설명
- 이용 방법
- 운영 시간
- 외부 링크 URL

#### 문의하기
- 안내 문구
- 문의 폼
- 개인정보 수집 동의 내용
- 연락처 정보

## 5. 디자인 요구사항

### 5.1 디자인 원칙
- **전문성**: B2B 고객에게 신뢰감을 주는 전문적인 디자인
- **명확성**: 정보 계층 구조가 명확한 레이아웃
- **일관성**: 모든 페이지에서 일관된 디자인 시스템
- **현대성**: 최신 웹 디자인 트렌드 반영

### 5.2 디자인 시스템

#### 컬러 팔레트
- 주 색상 (Primary): 브랜드 컬러 (1개)
- 보조 색상 (Secondary): 강조 색상 (1-2개)
- 중립 색상 (Neutral): 회색 계열 (3-4단계)
- 시스템 색상: 성공, 경고, 오류

**디자인 규칙**: 하나의 포인트 컬러를 기준으로 테마 적용 (다양한 컬러 활용 지양)

#### 타이포그래피
- 제목 폰트: 고딕 계열 (굵은 웨이트)
- 본문 폰트: 고딕 계열 (일반 웨이트)
- 폰트 크기: 모바일 최소 14px, 데스크톱 16px

#### 컴포넌트
- 버튼: Primary, Secondary, Outline
- 카드: 솔루션 카드, 시공사례 카드
- 폼 요소: Input, Textarea, Checkbox
- 아이콘: 일관된 스타일 (Line 또는 Solid)

### 5.3 레이아웃

#### 그리드 시스템
- 데스크톱: 12 컬럼
- 태블릿: 8 컬럼
- 모바일: 4 컬럼

#### 여백 (Spacing)
- 기본 단위: 8px
- 섹션 간 여백: 80px (데스크톱), 60px (모바일)
- 요소 간 여백: 16px, 24px, 32px

#### 최대 너비
- 컨텐츠 최대 너비: 1200px
- 텍스트 최대 너비: 800px (가독성)

### 5.4 공통 디자인 규칙
- 메인페이지와 서브페이지의 header와 footer는 공통 UI 활용
- 일관된 버튼 스타일 및 호버 효과
- 통일된 카드 디자인
- 명확한 시각적 계층 구조

## 6. 기술 요구사항

### 6.1 기술 스택

#### 프론트엔드
- HTML5 (시맨틱 태그)
- CSS3 (Flexbox, Grid, CSS Variables)
- JavaScript (ES6+)

#### 라이브러리 (CDN 방식)
- Swiper.js: 슬라이더
- GSAP: 애니메이션
- 카카오맵 API: 지도

#### 개발 규칙
- 외부 라이브러리는 CDN 방식 활용
- 애니메이션 관련 라이브러리는 GSAP로만 활용
- 이미지 들어가야 하는 공간은 더미 링크로 적용 (이미지만 넣으면 되는 구조)

- 페이지 로딩 속도: 2.5초 이내
- Lighthouse 점수: 90점 이상 (모든 항목)
- 이미지 최적화: WebP 형식, 레이지 로딩
- CSS/JS 압축 및 번들링

### 6.3 호환성 요구사항

#### 브라우저
- Chrome (최신 버전)
- Firefox (최신 버전)
- Safari (최신 버전)
- Edge (최신 버전)
- 모바일 브라우저 (iOS Safari, Chrome)

#### 디바이스
- 데스크톱: 1024px 이상
- 태블릿: 768px - 1023px
- 모바일: 320px - 767px

### 6.4 접근성 요구사항
- WCAG 2.1 AA 수준 준수
- 키보드 네비게이션 지원

- 색상 대비 4.5:1 이상
- 시맨틱 HTML 사용
- ARIA 레이블 적절히 사용

### 6.5 SEO 요구사항
