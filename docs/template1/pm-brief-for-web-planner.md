## 1. 프로젝트 배경 및 목표

### 1.1 회사 소개
- **회사명**: (주)아이티로그
- **사업 분야**: 건설 현장 제품 솔루션 설치 및 시공
- **주요 고객**: 건설사, 시공사, 공공기관
- **비즈니스 특성**: B2B, 고관여 의사결정, 신뢰 중심

### 1.2 프로젝트 목표
1. **비즈니스 목표**
   - 브랜드 신뢰도 향상
   - 온라인 문의 전환율 30% 증가
   - 정보 탐색 효율성 50% 향상
   - 모바일 사용자 경험 대폭 개선

2. **사용자 목표**
   - 필요한 정보를 빠르게 찾을 수 있는 직관적인 구조
   - 솔루션별 상세 정보 및 시공 사례 확인
   - 간편한 문의 및 원격지원 요청

3. **기술 목표**
   - 정적 웹사이트 (HTML/CSS/JavaScript)
   - 모바일 최적화 및 반응형 디자인
   - 접근성 준수 (WCAG 2.1 AA)
   - 빠른 페이지 로딩 속도 (3초 이내)

### 1.3 프로젝트 제약사항
- **정적 웹사이트**: 데이터베이스 없음, 서버 사이드 로직 없음
- **제외 기능**: 로그인, 회원가입, 게시판, 실시간 채팅
- **포함 기능**: 문의 폼, 검색, 다운로드, 지도 연동

   ```
   메인 페이지 진입 → 고객지원 메뉴 → 원격지원 클릭 → 
   외부 사이트 이동 (새 창)
   ```

   ```
   인증 및 특허 확인 → 고객사 확인 → 문의하기
   ```

   **시나리오 4: 현장 담당자 - 특정 솔루션 검색**
   ```
   메인 페이지 진입 → 검색 (예: "환경센서") →

   ```

## 4. 작업 가이드라인

### 4.1 정보 구조 설계 원칙
1. **사용자 중심**: 사용자의 멘탈 모델에 맞는 구조
2. **명확성**: 모호하지 않은 명확한 카테고리명
3. **일관성**: 전체 사이트에서 일관된 구조 및 네이밍
4. **확장성**: 향후 콘텐츠 추가를 고려한 유연한 구조
5. **접근성**: 모든 사용자가 쉽게 접근 가능한 구조

### 4.2 콘텐츠 전략 원칙
1. **신뢰 우선**: B2B 특성상 신뢰 구축이 최우선
2. **명확성**: 전문 용어 사용하되 이해하기 쉽게 설명
3. **데이터 기반**: 구체적인 수치 및 사례 제시
4. **행동 유도**: 각 페이지마다 명확한 CTA
5. **SEO 최적화**: 검색 엔진 친화적인 콘텐츠 구조

### 4.3 기능 명세 작성 원칙
1. **구체성**: 모호하지 않은 명확한 기능 정의
2. **실행 가능성**: 정적 웹사이트 제약 내에서 구현 가능한 기능
3. **우선순위**: Must Have, Should Have, Nice to Have 구분
4. **예외 처리**: 모든 예외 상황 및 에러 케이스 정의
5. **접근성**: WCAG 2.1 AA 준수를 위한 기능 요구사항

### 4.4 사용자 플로우 설계 원칙
1. **효율성**: 목표 달성까지의 단계 최소화
2. **명확성**: 각 단계에서 다음 행동이 명확함
3. **유연성**: 다양한 사용자 경로 허용
4. **피드백**: 각 단계에서 명확한 피드백 제공
5. **복구 가능성**: 실수 시 쉽게 복구 가능
