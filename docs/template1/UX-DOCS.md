## 1. 디자인 컨셉 개요

### 1.1 디자인 철학

**"Smart Simplicity"**

건설 현장 안전 솔루션의 전문성을 유지하면서도, 현대적이고 깔끔한 미니멀리즘을 추구합니다. 
복잡한 기술을 단순하고 직관적으로 전달하는 것이 핵심입니다.

### 1.2 핵심 키워드

- **Trendy (트렌디)**: 최신 웹 디자인 트렌드 반영
- **Simple (심플)**: 불필요한 요소 제거, 본질에 집중
- **Clean (깔끔)**: 여백을 활용한 시원한 레이아웃
- **Modern (현대적)**: 세련된 타이포그래피와 그래픽
- **Professional (전문적)**: B2B 신뢰감 유지

### 1.3 디자인 방향성

**Template1과의 차별화**
- Template1: 전통적이고 안정적인 기업 이미지
- Template2: 혁신적이고 스마트한 기술 기업 이미지

**트렌디한 요소**
- 대담한 타이포그래피
- 그라데이션 및 글래스모피즘 효과
- 마이크로 인터랙션
- 다이나믹한 애니메이션

**심플한 요소**
- 넉넉한 여백 (화이트 스페이스)
- 단일 포인트 컬러 시스템
- 미니멀한 아이콘
- 명확한 정보 계층

## 2. 무드보드 및 비주얼 레퍼런스

### 2.1 컬러 무드

**주요 컬러 방향**
- 차갑고 기술적인 느낌: 블루 계열
- 신뢰와 안정감: 다크 네이비 또는 차콜
- 포인트: 비비드한 시안 또는 일렉트릭 블루

**분위기**
- 깨끗하고 프로페셔널
- 미래지향적이고 스마트
- 차분하면서도 역동적

### 2.2 타이포그래피 무드

**스타일**
- 기하학적이고 모던한 산세리프
- 굵은 웨이트로 임팩트 있는 제목
- 적절한 자간과 행간으로 가독성 확보

**레퍼런스**
- Pretendard (한글): 모던하고 깔끔한 고딕
- Inter, Poppins (영문): 기하학적 산세리프

### 2.3 레이아웃 무드

**스타일**
- 비대칭 그리드로 역동성 표현
- 큰 이미지와 여백의 대비
- 카드 기반 모듈형 디자인
- 스크롤 기반 스토리텔링

**레퍼런스**
- Apple 제품 페이지: 미니멀하고 임팩트 있는 레이아웃
- Stripe: 깔끔한 B2B 웹사이트
- Linear: 모던한 SaaS 디자인

### 2.4 인터랙션 무드

**스타일**
- 부드럽고 자연스러운 애니메이션
- 호버 시 미묘한 변화 (확대, 그림자, 색상)
- 스크롤 기반 페이드인 효과
- 마이크로 인터랙션 (버튼, 카드)

## 3. 사용자 경험 전략

### 3.1 사용자 여정 맵

#### 시나리오 1: 솔루션 탐색 (현장 담당자)
```
진입 → 히어로 섹션 (3초 이내 핵심 메시지 파악)
     ↓
     솔루션 카드 (관심 솔루션 클릭)
     ↓
     솔루션 상세 (시스템 구성도 확인)
     ↓
     시공사례 (실제 적용 사례 확인)
     ↓
     문의하기 (간편한 폼 작성)
```

**UX 목표**: 3클릭 이내 원하는 정보 도달

#### 시나리오 2: 회사 신뢰도 확인 (의사결정자)
```
진입 → 히어로 섹션 (브랜드 슬로건)
     ↓
     숫자로 보는 신뢰 (실적 확인)
     ↓
     고객사 섹션 (주요 고객사 확인)
     ↓
     회사소개 (인증 및 특허)
     ↓
     문의하기
```

**UX 목표**: 신뢰 요소를 빠르게 확인

### 3.2 정보 아키텍처 원칙

**명확성 (Clarity)**
- 메뉴 구조 단순화 (최대 2단계)
- 명확한 라벨링
- 시각적 계층 구조

**일관성 (Consistency)**
- 모든 페이지에서 동일한 패턴
- 통일된 컴포넌트 사용
- 예측 가능한 인터랙션

**효율성 (Efficiency)**
- 빠른 로딩 속도
- 최소한의 클릭
- 명확한 CTA

### 3.3 접근성 전략

**WCAG 2.1 AA 준수**
- 색상 대비 4.5:1 이상
- 키보드 네비게이션 지원
- 스크린 리더 호환
- 명확한 포커스 표시

**모바일 우선**
- 터치 타겟 최소 44x44px
- 스와이프 제스처 지원
- 반응형 타이포그래피

## 4. 디자인 차별화 전략

### 4.1 Template1 vs Template2

| 요소 | Template1 | Template2 |
|------|-----------|-----------|
| **컬러** | 따뜻한 톤 (오렌지, 레드) | 차가운 톤 (블루, 시안) |
| **타이포** | 전통적 고딕 | 모던 기하학 산세리프 |
| **레이아웃** | 대칭적, 안정적 | 비대칭적, 역동적 |
| **여백** | 적당한 여백 | 넉넉한 여백 (미니멀) |
| **이미지** | 사실적 사진 | 일러스트 + 사진 혼합 |
| **애니메이션** | 기본적 | 다이나믹하고 세련됨 |
| **분위기** | 신뢰감, 안정감 | 혁신적, 스마트 |

### 4.2 경쟁사 대비 차별화

**일반적인 건설 안전 웹사이트**
- 복잡한 정보 나열
- 오래된 디자인
- 느린 로딩 속도

**Template2의 차별점**
- 핵심 정보만 간결하게
- 최신 디자인 트렌드 반영
- 빠르고 부드러운 경험
- 스토리텔링 중심

---

## 5. 브랜드 아이덴티티

### 5.1 브랜드 가치 시각화

**안전 (Safety)**
- 시각적 요소: 방패 아이콘, 체크 마크
- 컬러: 신뢰감 있는 블루
- 메시지: "사고 제로(Zero)"

**기술 (Technology)**
- 시각적 요소: AI 아이콘, 회로 패턴
- 컬러: 미래적인 시안
- 메시지: "스마트하게"

**신뢰 (Trust)**
- 시각적 요소: 숫자, 인증 마크
- 컬러: 안정적인 다크 톤
- 메시지: "20년 경험"

### 5.2 슬로건 시각화

**"건설은 안전하게! 안전은 스마트하게!"**

**타이포그래피 처리**
- "안전하게" / "스마트하게": 굵은 웨이트, 포인트 컬러
- 느낌표: 역동성 표현
- 큰 사이즈로 임팩트

**배경 처리**
- 건설 현장 이미지 (블러 처리)
- 그라데이션 오버레이
- 다이나믹한 그래픽 요소

---

## 6. 감성 및 톤앤매너

### 6.1 시각적 톤앤매너

**전문적이지만 접근하기 쉬운**
- 깔끔한 레이아웃
- 명확한 정보 전달
- 친근한 아이콘과 일러스트

**혁신적이지만 신뢰할 수 있는**
- 최신 디자인 트렌드
- 검증된 실적 강조
- 안정적인 컬러 팔레트

**기술적이지만 이해하기 쉬운**
- 시각화된 시스템 구성도
- 간결한 설명
- 직관적인 아이콘

### 6.2 인터랙션 톤앤매너

**부드럽고 자연스러운**
- Ease-out 애니메이션
- 0.3-0.5초 지속 시간
- 과하지 않은 효과

**반응적이고 즉각적인**
- 호버 시 즉시 피드백
- 클릭 시 명확한 반응
- 로딩 상태 표시

## 7. 디자인 원칙

### 7.1 핵심 원칙

**1. Less is More (덜어내기)**
- 불필요한 요소 제거
- 핵심 메시지에 집중
- 여백을 디자인 요소로 활용

**2. Content First (콘텐츠 우선)**
- 콘텐츠가 디자인을 이끔
- 가독성 최우선
- 명확한 정보 계층

**3. Mobile First (모바일 우선)**
- 모바일 경험 먼저 설계
- 점진적 향상 (Progressive Enhancement)
- 터치 친화적 인터페이스

**4. Performance Matters (성능 중요)**
- 빠른 로딩 속도
- 최적화된 이미지
- 효율적인 애니메이션

### 7.2 디자인 제약사항

**해야 할 것 (Do)**
- 넉넉한 여백 사용
- 단일 포인트 컬러 활용
- 명확한 CTA 버튼
- 일관된 컴포넌트 사용
- 반응형 디자인

**하지 말아야 할 것 (Don't)**
- 과도한 색상 사용
- 복잡한 레이아웃
- 느린 애니메이션
- 작은 터치 타겟
- 낮은 색상 대비

---

## 8. 페이지별 디자인 컨셉

### 8.1 메인 페이지

**히어로 섹션**
- 전체 화면 높이 (100vh)
- 대담한 타이포그래피
- 다이나믹한 배경 (그라데이션 또는 영상)
- 명확한 CTA 버튼 2개

**솔루션 섹션**
- 4개 카드 그리드
- 호버 시 부드러운 확대 효과
- 아이콘 + 제목 + 설명 + 링크
- 스크롤 애니메이션

**신뢰 섹션**
- 큰 숫자 강조
- 카운트업 애니메이션
- 미니멀한 레이아웃

### 8.2 솔루션 페이지

**헤더 섹션**
- 큰 제목
- 간결한 소개
- 배경 이미지 또는 그라데이션

**시스템 구성도**
- 시각화된 다이어그램
- 클릭 시 확대 보기
- 각 요소 설명

**시공사례**
- 이미지 중심 레이아웃
- 호버 시 정보 표시
- 그리드 배치

### 8.3 회사소개 페이지

**회사연혁**
- 세로 타임라인 (모바일)
- 좌우 지그재그 (데스크톱)
- 스크롤 애니메이션

**인증 및 특허**
- 그리드 레이아웃
- 이미지 중심
- 호버 시 확대

---

## 9. 인터랙션 디자인

### 9.1 마이크로 인터랙션

**버튼**
- 호버: 배경색 변화 + 약간 확대 (scale: 1.05)
- 클릭: 눌림 효과 (scale: 0.95)
- 지속 시간: 0.3초

**카드**
- 호버: 그림자 증가 + 약간 상승 (translateY: -8px)
- 지속 시간: 0.4초

**링크**
- 호버: 밑줄 애니메이션 (왼쪽에서 오른쪽)
- 지속 시간: 0.3초

### 9.2 페이지 전환

**스크롤 애니메이션**
- 페이드인 + 슬라이드업
- Intersection Observer 활용
- 한 번만 실행

**슬라이더**
- 부드러운 페이드 또는 슬라이드
- 5초 자동 재생
- 스와이프 제스처 지원

## 1. 디자인 시스템 개요

### 1.1 목적

일관되고 확장 가능한 디자인 언어를 구축하여:
- 모든 페이지에서 통일된 사용자 경험 제공
- 개발 효율성 향상
- 유지보수 용이성 확보
- 브랜드 아이덴티티 강화

### 1.2 원칙

**단순성 (Simplicity)**
- 최소한의 요소로 최대 효과
- 명확한 시각적 계층

**일관성 (Consistency)**
- 재사용 가능한 컴포넌트
- 통일된 패턴

**확장성 (Scalability)**
- 새로운 페이지 추가 용이
- 유연한 그리드 시스템

## 4. 스페이싱 시스템

### 4.1 기본 단위

**8px 기반 시스템**
```
모든 여백과 패딩은 8의 배수 사용
- 4px: 매우 작은 간격
- 8px: 작은 간격
- 16px: 기본 간격
- 24px: 중간 간격
- 32px: 큰 간격
- 48px: 매우 큰 간격
- 64px: 섹션 간격
- 80px: 대형 섹션 간격
- 120px: 특대 섹션 간격
```

### 4.2 CSS Variables

```css
:root {
  /* Spacing Scale */
  --space-xs: 0.25rem;   /* 4px */
  --space-sm: 0.5rem;    /* 8px */
  --space-md: 1rem;      /* 16px */
  --space-lg: 1.5rem;    /* 24px */
  --space-xl: 2rem;      /* 32px */
  --space-2xl: 3rem;     /* 48px */
  --space-3xl: 4rem;     /* 64px */
  --space-4xl: 5rem;     /* 80px */
  --space-5xl: 7.5rem;   /* 120px */
  
  /* Section Spacing */
  --section-padding-desktop: 5rem;  /* 80px */
  --section-padding-mobile: 3rem;   /* 48px */
  
  /* Container Padding */
  --container-padding-desktop: 2rem; /* 32px */
  --container-padding-mobile: 1rem;  /* 16px */
}
```

### 4.3 사용 예시

```css

section {
  padding: var(--section-padding-desktop) 0;
}

/* 카드 내부 패딩 */

  padding: var(--space-xl);
}

/* 요소 간 간격 */
.element + .element {
  margin-top: var(--space-lg);
}
```

## 1. 반응형 전략

### 1.1 모바일 퍼스트 접근

**원칙**:
- 모바일 화면을 기준으로 디자인 시작
- 점진적 향상 (Progressive Enhancement)
- 필수 콘텐츠 우선 배치
- 터치 친화적 인터페이스

**이유**:
- 모바일 트래픽 55% 이상 예상
- 현장 담당자의 모바일 사용 빈번
- 성능 최적화 용이

### 1.2 브레이크포인트

```css
/* Mobile First */
/* 기본: 320px - 767px (모바일) */

/* Tablet */
@media (min-width: 768px) {
  /* 768px - 1023px */
}

/* Desktop */
@media (min-width: 1024px) {
  /* 1024px 이상 */
}

/* Large Desktop (선택) */
@media (min-width: 1440px) {
  /* 1440px 이상 */
}
```

### 1.3 디바이스 타겟

**모바일 (320-767px)**
- iPhone SE (375px)
- iPhone 12/13/14 (390px)
- Samsung Galaxy (360px, 412px)

**태블릿 (768-1023px)**
- iPad (768px, 820px)
- iPad Pro (1024px)

**데스크톱 (1024px+)**
- 일반 노트북 (1366px, 1440px)
- 데스크톱 (1920px)

```css
:root {
  --color-primary: #0066FF;
```

```css
/* Mobile First */
.container {

}

  }
}
```
