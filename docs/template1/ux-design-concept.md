# UX 디자인 컨셉 및 방향성

## 1. 디자인 철학

### 핵심 가치
**"신뢰와 전문성을 시각화하다"**

공사/현장 솔루션 회사로서 안전성, 신뢰성, 전문성을 시각적으로 전달하며, 동시에 현대적이고 접근하기 쉬운 디지털 경험을 제공합니다.

### 디자인 원칙

#### 1.1 Professional & Trustworthy (전문적이고 신뢰할 수 있는)
- 깔끔하고 정돈된 레이아웃
- 산업 특성을 반영한 견고한 비주얼
- 실제 현장 이미지와 데이터 기반 콘텐츠
- 명확한 정보 전달과 투명성

#### 1.2 Modern & Innovative (현대적이고 혁신적인)
- 최신 웹 디자인 트렌드 반영
- 부드러운 애니메이션과 인터랙션
- 깔끔한 타이포그래피와 여백 활용
- 기술력을 보여주는 시각적 요소

#### 1.3 Clear & Efficient (명확하고 효율적인)
- 직관적인 정보 구조
- 빠른 정보 접근성
- 명확한 행동 유도 (CTA)
- 불필요한 요소 제거

#### 1.4 Accessible & Responsive (접근 가능하고 반응형)
- 모든 디바이스에서 최적화된 경험
- 접근성 표준 준수
- 다양한 사용자 환경 고려
- 빠른 로딩 속도

## 2. 타겟 사용자 분석

### 2.1 주요 페르소나

#### 페르소나 1: 건설사 구매 담당자 (40대, 남성)
- **목표**: 신뢰할 수 있는 솔루션 공급업체 찾기
- **니즈**: 제품 사양, 실적, 가격 정보
- **행동**: 여러 업체 비교 검토, 상세 정보 다운로드
- **페인포인트**: 정보 부족, 연락 어려움, 불명확한 가격

#### 페르소나 2: 현장 관리자 (30-40대, 남/여)
- **목표**: 현장에 적합한 솔루션 확인
- **니즈**: 설치 사례, 기술 지원, 빠른 문의
- **행동**: 모바일로 빠른 정보 확인, 전화 문의
- **페인포인트**: 복잡한 웹사이트, 느린 로딩

#### 페르소나 3: 경영진/의사결정권자 (50대, 남성)
- **목표**: 회사 신뢰도 및 규모 파악
- **니즈**: 회사 소개, 주요 실적, 인증서
- **행동**: 간략한 정보 확인, 대표 연락처
- **페인포인트**: 핵심 정보 찾기 어려움

### 2.2 사용자 여정 맵

#### 인지 단계 (Awareness)
- 검색 엔진을 통한 유입
- 추천/소개를 통한 방문
- **제공 가치**: 명확한 첫인상, 전문성 전달

#### 고려 단계 (Consideration)
- 제품/서비스 상세 정보 탐색
- 실적 및 사례 확인
- 경쟁사와 비교
- **제공 가치**: 상세하고 신뢰할 수 있는 정보

#### 결정 단계 (Decision)
- 견적 문의
- 상담 신청
- 자료 다운로드
- **제공 가치**: 쉽고 빠른 문의 프로세스

#### 유지 단계 (Retention)
- 사후 지원 정보 확인
- 추가 제품 탐색
- **제공 가치**: 지속적인 관계 유지

## 3. 디자인 컨셉

### 3.1 비주얼 컨셉: "Solid Foundation"

**컨셉 설명**
건설/공사 현장의 견고함과 안정성을 현대적인 디자인 언어로 재해석합니다. 
구조적인 그리드 시스템, 명확한 타이포그래피, 산업적 색상 팔레트를 통해 
전문성과 신뢰감을 전달하면서도 접근하기 쉬운 디지털 경험을 제공합니다.

### 3.2 무드보드 키워드
- **산업적 (Industrial)**: 강철, 콘크리트, 구조물
- **현대적 (Modern)**: 미니멀, 깔끔함, 여백
- **기술적 (Technical)**: 정밀함, 데이터, 시스템
- **신뢰감 (Trustworthy)**: 안정적, 전문적, 투명함

### 3.3 디자인 모티브
- **그리드와 구조**: 건축 도면의 그리드 시스템을 디자인에 반영
- **레이어링**: 현장의 층층이 쌓이는 구조를 시각적으로 표현
- **정밀함**: 측정과 정확성을 나타내는 디테일
- **실사 이미지**: 실제 현장과 제품의 생생한 이미지 활용

## 4. 색상 전략

### 4.1 Primary Color Palette

#### 메인 컬러: Industrial Blue
```
Primary: #1A4D8F (진한 파랑)
- 신뢰, 전문성, 안정성을 상징
- 산업 분야에서 보편적으로 신뢰받는 색상
- 헤더, 주요 CTA, 강조 요소에 사용
```

#### 보조 컬러: Steel Gray
```
Secondary: #5A6C7D (스틸 그레이)
- 산업적 느낌과 현대적 감각
- 텍스트, 아이콘, 보조 요소에 사용
- 메인 컬러와 조화로운 중립적 톤
```

#### 강조 컬러: Safety Orange
```
Accent: #FF6B35 (안전 오렌지)
- 주목성과 행동 유도
- 현장 안전 표지의 오렌지 색상 연상
- CTA 버튼, 중요 알림, 하이라이트에 사용
```

### 4.2 Neutral Colors
```
Background White: #FFFFFF
Background Light: #F8F9FA
Background Gray: #E9ECEF
Border Gray: #DEE2E6

Text Primary: #212529
Text Secondary: #6C757D
Text Light: #ADB5BD
```

### 4.3 Semantic Colors
```
Success: #28A745 (완료, 성공)
Warning: #FFC107 (주의, 경고)
Error: #DC3545 (오류, 위험)
Info: #17A2B8 (정보, 안내)
```

### 4.4 색상 사용 원칙
- **70-20-10 규칙**: 중립색 70%, 메인 컬러 20%, 강조 컬러 10%
- **접근성**: 모든 텍스트는 배경과 4.5:1 이상의 대비율 유지
- **일관성**: 동일한 의미는 동일한 색상으로 표현
- **계층**: 색상 농도로 정보의 중요도 표현

## 5. 타이포그래피 전략

### 5.1 폰트 선택

#### 한글 폰트: Pretendard
```
- 현대적이고 가독성 높은 한글 폰트
- 다양한 굵기 지원 (100-900)
- 웹 최적화 및 무료 사용 가능
- 기술 문서에 적합한 명확한 형태
```

#### 영문/숫자 폰트: Inter
```
- 디지털 환경에 최적화된 폰트
- 숫자 가독성 우수 (데이터 표현에 적합)
- Pretendard와 조화로운 디자인
- Variable Font 지원으로 성능 최적화
```

#### 강조 폰트: Montserrat (선택적)
```
- 헤드라인, 숫자 강조용
- 기하학적이고 현대적인 느낌
- 굵은 굵기(700-800)로 임팩트 있는 표현
```

### 5.2 타이포그래피 스케일

#### 데스크톱
```
H1 (Hero): 56px / Bold (700) / Line-height 1.2
H2 (Section): 40px / Bold (700) / Line-height 1.3
H3 (Subsection): 32px / SemiBold (600) / Line-height 1.4
H4 (Card Title): 24px / SemiBold (600) / Line-height 1.5
H5 (Small Title): 20px / Medium (500) / Line-height 1.5
Body Large: 18px / Regular (400) / Line-height 1.7
Body: 16px / Regular (400) / Line-height 1.6
Body Small: 14px / Regular (400) / Line-height 1.6
Caption: 12px / Regular (400) / Line-height 1.5
```

#### 모바일
```
H1 (Hero): 40px / Bold (700) / Line-height 1.2
H2 (Section): 32px / Bold (700) / Line-height 1.3
H3 (Subsection): 24px / SemiBold (600) / Line-height 1.4
H4 (Card Title): 20px / SemiBold (600) / Line-height 1.5
Body: 16px / Regular (400) / Line-height 1.6
Body Small: 14px / Regular (400) / Line-height 1.6
Caption: 12px / Regular (400) / Line-height 1.5
```

### 5.3 타이포그래피 원칙
- **가독성 우선**: 충분한 행간(1.6-1.7)과 자간
- **계층 구조**: 크기, 굵기, 색상으로 명확한 위계 설정
- **일관성**: 동일한 레벨은 동일한 스타일 적용
- **반응형**: 모바일에서 적절한 크기 조정

## 6. 이미지 및 비주얼 스타일

### 6.1 사진 스타일

#### 히어로 이미지
- **스타일**: 실제 현장의 다이나믹한 장면
- **구도**: 와이드 앵글, 깊이감 있는 구도
- **색감**: 자연스러운 색상, 약간의 대비 강조
- **오버레이**: 어두운 그라데이션으로 텍스트 가독성 확보

#### 제품 이미지
- **스타일**: 깔끔한 배경, 제품 중심
- **조명**: 균일하고 전문적인 조명
- **앵글**: 제품 특징이 잘 보이는 각도
- **배경**: 흰색 또는 연한 회색 배경

#### 현장 사례 이미지
- **스타일**: 실제 설치 현장의 생생한 모습
- **구도**: Before/After 비교 가능한 구도
- **품질**: 고해상도, 선명한 이미지
- **다양성**: 다양한 현장 유형 포함

### 6.2 아이콘 스타일
- **스타일**: Line icon (선 굵기 2px)
- **형태**: 기하학적이고 명확한 형태
- **크기**: 24px, 32px, 48px (용도별)
- **색상**: 단색 또는 Primary Color
- **일관성**: 동일한 아이콘 세트 사용 (예: Feather Icons, Heroicons)

### 6.3 일러스트레이션
- **스타일**: 미니멀하고 기하학적
- **용도**: 빈 상태, 에러 페이지, 프로세스 설명
- **색상**: 브랜드 컬러 팔레트 내에서 사용
- **톤**: 전문적이면서도 친근한 느낌

### 6.4 그래픽 요소
- **그리드 패턴**: 배경에 은은한 그리드 라인
- **기하학적 도형**: 섹션 구분, 강조 요소
- **그라데이션**: 미묘한 그라데이션 (과도하지 않게)
- **라인**: 구분선, 장식 요소로 활용

## 7. 인터랙션 및 애니메이션

### 7.1 마이크로 인터랙션

#### 버튼 호버
```
- Transform: scale(1.02) + 그림자 증가
- Duration: 0.2s
- Easing: ease-out
```

#### 카드 호버
```
- Transform: translateY(-4px) + 그림자 증가
- Duration: 0.3s
- Easing: ease-out
```

#### 링크 호버
```
- 밑줄 애니메이션 (왼쪽에서 오른쪽)
- 색상 변화
- Duration: 0.2s
```

### 7.2 페이지 전환
- **스크롤 애니메이션**: Fade-in + Slide-up (요소가 뷰포트에 들어올 때)
- **페이지 로드**: 부드러운 Fade-in
- **이미지 로드**: Progressive loading + Blur-up 효과

### 7.3 애니메이션 원칙
- **성능 우선**: GPU 가속 속성만 사용 (transform, opacity)
- **자연스러움**: Ease-out, ease-in-out 사용
- **빠른 반응**: 0.2-0.3초 이내
- **접근성**: prefers-reduced-motion 고려

## 8. 반응형 디자인 전략

### 8.1 모바일 퍼스트 접근
- 모바일 화면을 기준으로 설계 시작
- 점진적 향상 (Progressive Enhancement)
- 터치 인터랙션 최적화
- 최소 터치 영역: 44x44px

### 8.2 브레이크포인트 전략
```
Mobile: 320px - 767px (1단 레이아웃)
Tablet: 768px - 1023px (2단 레이아웃)
Desktop: 1024px - 1439px (3-4단 레이아웃)
Large Desktop: 1440px+ (최대 너비 제한)
```

### 8.3 반응형 패턴
- **네비게이션**: 모바일 햄버거 메뉴 → 데스크톱 풀 메뉴
- **그리드**: 1단 → 2단 → 3-4단
- **이미지**: 반응형 이미지 (srcset, picture 태그)
- **타이포그래피**: 유동적 크기 조정

## 9. 접근성 고려사항

### 9.1 WCAG 2.1 AA 준수
- 색상 대비: 최소 4.5:1 (일반 텍스트), 3:1 (큰 텍스트)
- 키보드 접근: 모든 기능 키보드로 조작 가능
- 포커스 표시: 명확한 포커스 인디케이터
- 대체 텍스트: 모든 이미지에 의미 있는 alt 제공

### 9.2 스크린 리더 지원
- 시맨틱 HTML 사용
- ARIA 레이블 적절히 사용
- 랜드마크 역할 정의
- 건너뛰기 링크 제공

### 9.3 다양한 사용자 고려
- 확대/축소 지원 (최대 200%)
- 충분한 터치 영역
- 명확한 에러 메시지
- 자동 재생 비디오 제어 가능

## 10. 차별화 포인트

### 10.1 경쟁사 대비 차별점
- **현대적 디자인**: 전통적인 산업 사이트와 차별화된 모던한 UI
- **사용자 중심**: 복잡한 정보를 직관적으로 전달
- **모바일 최적화**: 현장에서 바로 확인 가능한 모바일 경험
- **신뢰 구축**: 실제 데이터와 사례 중심의 콘텐츠

### 10.2 핵심 경험 요소
- **빠른 정보 접근**: 3클릭 이내 원하는 정보 도달
- **명확한 CTA**: 각 페이지마다 명확한 다음 행동 제시
- **실시간 문의**: 쉽고 빠른 문의 채널
- **신뢰 지표**: 인증서, 실적, 고객사 로고 등 가시화

## 11. 성공 지표

### 11.1 UX 지표
- 평균 세션 시간: 3분 이상
- 이탈률: 40% 이하
- 페이지뷰/세션: 4페이지 이상
- 모바일 이탈률: 50% 이하

### 11.2 비즈니스 지표
- 문의 전환율: 5% 이상
- 자료 다운로드: 월 100건 이상
- 재방문율: 30% 이상
- 평균 체류 시간: 2분 이상

### 11.3 기술 지표
- Lighthouse 성능 점수: 90점 이상
- 접근성 점수: 95점 이상
- 페이지 로딩 속도: 2초 이내
- Core Web Vitals: 모두 Good 등급
