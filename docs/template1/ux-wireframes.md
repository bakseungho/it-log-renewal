# UX 와이어프레임 설명

## 1. 와이어프레임 개요

### 1.1 목적
이 문서는 공사/현장 솔루션 회사 홈페이지의 주요 페이지별 와이어프레임을 텍스트와 ASCII 아트로 설명합니다. 실제 디자인 작업 시 Figma 등의 도구로 시각화할 기준을 제공합니다.

### 1.2 표기 규칙
```
[요소명]: 인터랙티브 요소 (버튼, 링크 등)
{콘텐츠}: 텍스트 또는 이미지 콘텐츠
---: 구분선
│: 세로 구분
```

### 1.3 반응형 접근
- 데스크톱 (1200px+): 기본 레이아웃
- 태블릿 (768-1023px): 2단 레이아웃
- 모바일 (320-767px): 1단 레이아웃

## 9. 인터랙션 상세

### 9.1 페이지 로드 애니메이션

```
1. 헤더: Fade in (0.3s)
2. Hero 섹션: Fade in + Slide up (0.5s, delay 0.2s)
3. 콘텐츠 섹션: Fade in + Slide up (0.4s, 순차적으로)
```

### 9.2 스크롤 애니메이션

```
요소가 뷰포트에 진입할 때:
- Fade in (투명도 0 → 1)
- Slide up (Y축 20px 이동)
- Duration: 0.6s
- Easing: ease-out
```

### 9.3 호버 상태

```
카드 호버:
- Transform: translateY(-8px)
- Box-shadow: 증가
- Duration: 0.3s

버튼 호버:
- Background: 10% 어둡게
- Transform: scale(1.02)
- Duration: 0.2s

링크 호버:
- 밑줄 애니메이션 (왼쪽 → 오른쪽)
- Color: Primary → Accent
- Duration: 0.2s
```

### 9.4 모달/오버레이

```
열기:
- 배경: Fade in (0.3s)
- 콘텐츠: Scale up (0.9 → 1) + Fade in (0.3s)

닫기:
- 콘텐츠: Scale down (1 → 0.9) + Fade out (0.2s)
- 배경: Fade out (0.3s)
```

```
포커스 표시:
┌──────────────────┐
│  버튼 텍스트     │  ← 2px solid Primary Color 아웃라인

```

### 10.2 스크린 리더 지원
