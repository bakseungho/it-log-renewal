# 빠른 참조 가이드 (Quick Reference)

> 프로젝트 팀원들이 필요한 문서를 빠르게 찾을 수 있도록 정리한 가이드입니다.

---

## 🚀 시작하기

### 처음 프로젝트에 참여하셨나요?

**1단계**: 역할 확인
- [ ] 웹 기획자
- [ ] UX 디자이너  
- [ ] 프론트엔드 개발자
- [ ] PM

**2단계**: 해당 역할의 "시작 문서" 읽기 (아래 참조)

**3단계**: 작업 지시서 기반으로 작업 시작

---

## 📖 역할별 시작 문서

### 👨‍💼 PM (Product Manager)

**지금 바로 읽어야 할 문서 (순서대로)**:
1. `pm-project-brief.md` - 프로젝트 전체 개요 (30분)
2. `pm-task-assignment.md` - 작업 배분 및 일정 (20분)
3. `hopage_renewal_content.md` - 콘텐츠 요구사항 (40분)

**총 소요 시간**: 약 1.5시간

---

### 📝 웹 기획자 (Week 1-2)

**지금 바로 읽어야 할 문서 (순서대로)**:
1. ⭐ `pm-brief-for-web-planner.md` - **작업 지시서** (1시간)
2. `hopage_renewal_content.md` - 콘텐츠 요구사항 (40분)
3. `pm-project-brief.md` - 프로젝트 배경 (30분)

**참고 문서**:
- `website-renewal-guidelines.md` - 가이드라인

**산출물 제출**:
- Week 1 말: 사이트맵, IA, 콘텐츠 전략
- Week 2 말: 기능 명세서, 사용자 플로우

**총 소요 시간**: 약 2시간

---

### 🎨 UX 디자이너 (Week 3-4)

**지금 바로 읽어야 할 문서 (순서대로)**:
1. ⭐ `pm-brief-for-ux-designer.md` - **작업 지시서** (1시간)
2. `hopage_renewal_content.md` - 콘텐츠 요구사항 (40분)
3. 웹 기획자 산출물 (Week 2 말 인계) (30분)

**참고 문서**:
- `pm-requirements-for-ux.md` - 상세 요구사항 (선택)
- `website-renewal-guidelines.md` - 디자인 가이드라인
- `ux-design-concept.md` - 디자인 컨셉 참고

**산출물 제출**:
- Week 3 말: 와이어프레임
- Week 4 말: 프로토타입, 디자인 시스템

**총 소요 시간**: 약 2시간

---

### 💻 프론트엔드 개발자 (Week 5-8)

**지금 바로 읽어야 할 문서 (순서대로)**:
1. ⭐ `pm-brief-for-frontend-dev.md` - **작업 지시서** (1시간)
2. `frontend-technical-spec.md` - 기술 명세 (40분)
3. `DEVELOPMENT.md` - 개발 가이드 (30분)
4. UX 디자이너 산출물 (Week 4 말 인계) (30분)

**참고 문서**:
- `pm-requirements-for-frontend.md` - 상세 요구사항 (선택)
- `PROJECT-SUMMARY.md` - 프로젝트 구조
- `website-renewal-guidelines.md` - 기술 가이드라인

**산출물 제출**:
- Week 5 말: 공통 요소 + 메인 페이지
- Week 6 말: 서브 페이지
- Week 7 말: JavaScript 기능 + 최적화
- Week 8 말: 테스트 완료 + 배포

**총 소요 시간**: 약 2.5시간

## 🔍 상황별 문서 찾기

### "프로젝트 목표가 뭐였지?"
→ `pm-project-brief.md` 섹션 2

### "내 작업 일정이 언제까지지?"
→ `pm-task-assignment.md` 또는 해당 역할의 작업 지시서

### "이 페이지에 어떤 콘텐츠가 들어가지?"
→ `hopage_renewal_content.md`

### "디자인 컬러는 뭐로 하지?"
→ `ux-design-system.md` 또는 `website-renewal-guidelines.md`

### "기술 스택이 뭐였지?"
→ `frontend-technical-spec.md` 또는 `PROJECT-SUMMARY.md`

### "접근성 기준이 뭐지?"
→ `pm-requirements-for-ux.md` 섹션 7 또는 `website-renewal-guidelines.md`

### "이전 단계 산출물은 어디 있지?"
→ `docs/` 폴더에서 `ux-*.md` 또는 `web-planning-*.md` 검색

### "문의 폼 기능 명세는?"
→ `pm-brief-for-web-planner.md` 또는 웹 기획자 산출물

## ⚡ 긴급 상황별 대응

### "작업 지시서를 못 찾겠어요"
→ `docs/pm-brief-for-[역할].md` 파일 찾기
- 웹 기획자: `pm-brief-for-web-planner.md`
- UX 디자이너: `pm-brief-for-ux-designer.md`
- 프론트엔드: `pm-brief-for-frontend-dev.md`

### "문서가 너무 많아요"
→ 이 문서의 "역할별 시작 문서" 섹션만 보세요

### "문서 내용이 서로 달라요"
→ 작업 지시서 (`pm-brief-for-*.md`)가 최신 버전입니다
→ 충돌 시 PM에게 문의

### "이전 단계 산출물이 없어요"
→ PM에게 인계 일정 확인
→ 해당 역할 담당자에게 직접 요청

## 📞 도움 요청

### 문서 관련
- **찾을 수 없음**: PM에게 문의
