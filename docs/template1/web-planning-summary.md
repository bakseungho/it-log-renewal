## 2. 주요 발견 사항

### 2.1 콘텐츠 요구사항과 UX 문서 간 불일치

**문제점:**
`hopage_renewal_content.md`에 명시된 메뉴 구조와 `ux-information-architecture.md`의 메뉴 구조가 다릅니다.

**콘텐츠 문서 (실제 요구사항):**
```
1. Home (메인페이지)
2. 회사소개
   - 회사소개
   - CEO 인사말
   - 회사연혁
   - 인증 및 특허
   - 오시는 길
3. 현장 안전 플랫폼
   - 스마트 현장안전 시스템
   - 타워크레인 통합안전 시스템
4. 스마트 관제·제어
   - AI 영상 방송 관제시스템
5. 환경·출입관리
   - 스마트 환경센서 시스템
   - 스마트 출입통제 시스템
6. 고객지원
   - 원격지원
   - 문의하기
```

**UX 문서 (현재):**
```
1. 회사소개
2. 사업분야 (← 콘텐츠 문서에 없음)
3. 제품소개 (← 콘텐츠 문서에 없음)
4. 실적 및 사례 (← 콘텐츠 문서에 없음)
5. 지원센터
6. 채용정보 (← 콘텐츠 문서에 없음)
```

**권장 조치:**
- UX 정보 구조도를 콘텐츠 요구사항에 맞게 수정
- 또는 PM과 협의하여 최종 메뉴 구조 확정

---

### 2.2 솔루션 구조 차이

**콘텐츠 문서:**
- 솔루션을 3개 대분류로 구분 (현장 안전 플랫폼, 스마트 관제·제어, 환경·출입관리)
- 각 대분류 아래 2Depth 솔루션 페이지
- 각 솔루션 페이지 내에 하위 시스템 포함

**UX 문서:**
- 제품소개 단일 메뉴
- 카테고리별 제품 분류
- 제품 상세 페이지 구조

**권장 조치:**
- 콘텐츠 문서의 구조를 따르는 것이 명확함
- 웹 기획 산출물은 콘텐츠 문서 기준으로 작성 완료

## 3. 웹 기획 산출물 핵심 내용

### 3.1 사이트맵 (web-planning-sitemap.md)

**주요 내용:**
- 전체 23개 페이지 정의
- 1Depth 6개, 2Depth 17개
- URL 구조: `/about/`, `/solutions/`, `/support/`
- SEO 친화적 URL 네이밍
- 브레드크럼 구조 정의
- 모바일 네비게이션 구조

**핵심 결정사항:**
- 솔루션 페이지 URL: `/solutions/{solution-name}.html`
- 모든 URL 소문자, 하이픈 구분
- 정적 HTML 파일 (.html 확장자)

---

### 3.2 콘텐츠 전략 (web-planning-content-strategy.md)

**주요 내용:**
- 페이지별 콘텐츠 매트릭스 (목적, 타겟, 메시지, CTA)
- 히어로 섹션 4개 슬라이드 상세 문구
- 콘텐츠 톤앤매너 가이드
- SEO 키워드 전략 (페이지별 타겟 키워드)
- 콘텐츠 갭 분석 (추가 필요 콘텐츠 목록)
- 콘텐츠 수집 계획 (3주 일정)

**핵심 결정사항:**
- 톤앤매너: 전문적, 신뢰할 수 있는, 명확한, 현대적
- 주요 키워드: 건설 현장 안전, 스마트 현장 관리, AI 영상 관제 등
- 추가 필요 콘텐츠: 솔루션 상세 설명, 시공사례 이미지, 인증서 등

---

### 3.3 기능 명세서 (web-planning-functional-spec.md)

**주요 내용:**
- 공통 기능: 헤더, 푸터, 플로팅 버튼
- 메인 페이지 기능: 히어로 슬라이더, 솔루션 카드, 카운터 애니메이션
- 솔루션 페이지 기능: 이미지 갤러리, 라이트박스
- 회사소개 페이지 기능: 타임라인, 지도 API
- 고객지원 페이지 기능: 문의 폼, 입력 검증
- 검색 기능, 애니메이션, 반응형, 접근성

**핵심 결정사항:**
- 히어로 슬라이더: Swiper.js, 5초 자동 재생
- 문의 폼: FormSpree 또는 EmailJS 사용
- 지도: 카카오맵 API 우선
- 이미지 최적화: Lazy Loading, WebP 지원
- 접근성: WCAG 2.1 AA 준수

---

### 3.4 사용자 플로우 (web-planning-user-flow.md)

**주요 내용:**
- 4개 주요 시나리오 정의
  1. 신규 고객 - 솔루션 탐색 및 문의
  2. 기존 고객 - 원격지원 요청
  3. 구매 담당자 - 회사 신뢰도 확인
  4. 현장 담당자 - 특정 솔루션 검색
- 전환 퍼널 분석 (목표 전환율 5%)
- 단계별 이탈 방지 전략
- 사용자 여정 맵
- 모바일 사용자 플로우

**핵심 결정사항:**
- 주요 전환 목표: 문의 폼 제출, 전화 문의
- 이탈 방지: 명확한 CTA, 신뢰 요소 강화, 폼 간소화
- 모바일 최적화: 클릭투콜, 간소화된 네비게이션

## 5. 수정 및 보완 필요 사항

### 5.1 UX 정보 구조도 수정 필요

**파일**: `docs/ux-information-architecture.md`

**수정 사항:**
1. 메뉴 구조를 콘텐츠 문서에 맞게 변경
   - "사업분야" → "현장 안전 플랫폼", "스마트 관제·제어", "환경·출입관리"
   - "제품소개" 삭제
   - "실적 및 사례" → 각 솔루션 페이지 내 "주요 시공 사례" 섹션으로 통합
   - "채용정보" 삭제 (Phase 2 이후)

2. 솔루션 페이지 구조 변경
   - 3개 대분류 메뉴 (현장 안전 플랫폼, 스마트 관제·제어, 환경·출입관리)
   - 각 대분류 아래 2Depth 솔루션 페이지
   - 각 솔루션 페이지 내 하위 시스템 섹션

3. URL 구조 통일
   - `/solutions/{solution-name}.html` 형식 사용

---

### 5.2 UX 와이어프레임 수정 필요

**파일**: `docs/ux-wireframes.md`

**수정 사항:**
1. 헤더 네비게이션 메뉴명 변경
   - 콘텐츠 문서의 메뉴명 사용

2. 솔루션 페이지 와이어프레임 추가
   - 하위 시스템 표시 방법 (탭 또는 아코디언)
   - 시스템 구성도 섹션
   - 시공사례 섹션

3. 제품 상세 페이지 → 솔루션 상세 페이지로 변경
   - 제품 중심이 아닌 솔루션 중심 구조

## 7. 주요 의사결정 필요 사항

### 7.1 메뉴 구조 최종 확정

**옵션 A: 콘텐츠 문서 기준 (권장)**
```
1. 회사소개
2. 현장 안전 플랫폼
3. 스마트 관제·제어
4. 환경·출입관리
5. 고객지원
```

**장점:**
- 솔루션 중심 구조 (비즈니스 목표에 부합)
- 명확한 카테고리 구분
- 확장 가능한 구조

**단점:**
- 1Depth 메뉴가 5개로 많음 (모바일에서 복잡할 수 있음)

**옵션 B: UX 문서 기준**
```
1. 회사소개
2. 사업분야 (솔루션 통합)
3. 실적 및 사례
4. 지원센터
```

**장점:**
- 간결한 메뉴 구조
- 일반적인 B2B 웹사이트 구조

**단점:**
- 콘텐츠 요구사항과 불일치
- 솔루션 구분이 명확하지 않음

**권장**: 옵션 A (콘텐츠 문서 기준)

---

### 7.2 시공사례 표시 방법

**옵션 A: 각 솔루션 페이지 내 섹션 (권장)**
- 각 솔루션 페이지에 "주요 시공 사례" 섹션
- 해당 솔루션 관련 사례만 표시

**옵션 B: 별도 시공사례 페이지**
- 독립된 "실적 및 사례" 메뉴
- 모든 사례를 한 곳에 모아 표시
- 필터링 기능 제공

**권장**: 옵션 A (콘텐츠 문서 기준)

---
