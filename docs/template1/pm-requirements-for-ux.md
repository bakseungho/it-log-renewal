# UX 디자이너를 위한 요구사항 문서

## 문서 개요

**프로젝트명**: 홈페이지 리뉴얼 프로젝트  
**대상**: UX 디자이너  
**목적**: B2B 공사/현장 솔루션 회사의 정적 웹사이트 UX 설계 가이드  
**작성일**: 2024  
**버전**: 1.0

---

## 1. 프로젝트 컨텍스트

### 1.1 비즈니스 배경
- **회사 유형**: B2B 공사/현장 제품 솔루션 설치 전문 기업
- **주요 고객**: 건설사, 시공사, 공공기관, 산업체
- **비즈니스 특성**: 고관여 의사결정, 장기 검토 프로세스, 신뢰 중심
- **웹사이트 역할**: 브랜드 신뢰 구축, 정보 제공, 리드 생성

### 1.2 프로젝트 목표
- 전문적이고 신뢰할 수 있는 B2B 웹사이트 경험 제공
- 정보 탐색 효율성 50% 향상
- 문의 전환율 30% 증가
- 모바일 사용자 경험 대폭 개선

### 1.3 기술 제약사항
- **정적 웹사이트**: DB 없이 HTML/CSS/JavaScript만 사용
- **제외 기능**: 로그인, 회원가입, 게시판, 실시간 채팅
- **포함 기능**: 문의 폼, 검색, 다운로드, 지도 연동

- 20년 경력의 베테랑

**목표 및 니즈**

- 회사 실적과 기술력 확인

**행동 패턴**

**페인 포인트**

- 연락처 찾기 어려움

**UX 요구사항**
- 명확한 정보 구조
- 실적 및 인증 정보 전면 배치
- 다운로드 가능한 회사 소개서
- 명확한 연락처 및 CTA

- 전화 문의 선호

**페인 포인트**

**UX 요구사항**

**UX 요구사항**
- 간편한 문의 폼
- 명확한 제품 정보
- 빠른 응답 기대치 설정
- 자료 일괄 다운로드


### 2.2 사용자 여정 맵

#### 여정 1: 신규 고객 (정보 탐색)
```
인지 → 탐색 → 평가 → 문의 → 전환
```

**1단계: 인지 (Awareness)**
- 유입 경로: 검색 엔진, 추천, 전시회
- 사용자 생각: "이 회사가 우리 프로젝트에 적합할까?"
- 필요 정보: 회사 개요, 주요 제품/서비스
- UX 목표: 첫인상으로 신뢰 구축

**2단계: 탐색 (Exploration)**
- 행동: 메뉴 탐색, 제품 정보 확인
- 사용자 생각: "어떤 솔루션이 있고, 우리 현장에 맞을까?"
- 필요 정보: 제품 카테고리, 기술 사양, 적용 사례
- UX 목표: 직관적인 네비게이션, 빠른 정보 접근

**3단계: 평가 (Evaluation)**
- 행동: 실적 확인, 고객사 검토, 자료 다운로드
- 사용자 생각: "이 회사를 믿을 수 있을까? 실적은?"
- 필요 정보: 프로젝트 포트폴리오, 고객사, 인증서
- UX 목표: 신뢰 요소 강화, 상세 정보 제공

**4단계: 문의 (Contact)**
- 행동: 문의 폼 작성 또는 전화
- 사용자 생각: "빠르게 상담받고 싶다"
- 필요 정보: 연락처, 문의 방법
- UX 목표: 간편한 문의 프로세스

**5단계: 전환 (Conversion)**
- 행동: 견적 요청, 미팅 일정 조율
- 사용자 생각: "빠른 응답을 기대한다"
- 필요 정보: 응답 시간, 다음 단계
- UX 목표: 명확한 기대치 설정

## 3. 정보 구조 (Information Architecture)

### 3.1 사이트맵

```
홈
├── 회사 소개
│   ├── 회사 개요
│   ├── 연혁
│   ├── 비전 및 미션
│   ├── 인증 및 특허
│   └── 조직도
├── 제품/솔루션
│   ├── 제품 카테고리 A
│   ├── 제품 카테고리 B
│   ├── 제품 카테고리 C
│   └── 맞춤형 솔루션
├── 시공 실적
│   ├── 프로젝트 포트폴리오
│   ├── 주요 고객사
│   └── 현장 사진
├── 기술 지원
│   ├── FAQ
│   ├── 자료실 (카탈로그, 매뉴얼)
│   └── 기술 문서
└── 고객 지원
    ├── 문의하기
    ├── 견적 요청
    ├── 연락처
    └── 오시는 길
```

### 3.2 네비게이션 구조

#### 글로벌 네비게이션 (GNB)
- 위치: 상단 고정 (Sticky Header)
- 구성: 로고 + 메인 메뉴 + 검색 + 문의하기 버튼
- 메뉴 항목: 회사 소개 | 제품/솔루션 | 시공 실적 | 기술 지원 | 고객 지원
- 모바일: 햄버거 메뉴 (전체 화면 오버레이)

#### 서브 네비게이션
- 2depth 메뉴: 호버 시 드롭다운 (데스크톱)
- 모바일: 아코디언 방식
- 현재 위치 표시 (Breadcrumb)

#### 푸터 네비게이션
- 회사 정보 (주소, 전화, 이메일)
- 빠른 링크 (주요 페이지)
- SNS 링크
- 개인정보처리방침, 이용약관


### 3.3 콘텐츠 우선순위 (각 페이지별)

#### 메인 페이지
1. **히어로 섹션**: 핵심 메시지 + 주요 CTA
2. **회사 소개**: 간략한 소개 + 핵심 강점
3. **주요 제품/솔루션**: 3-4개 대표 제품
4. **시공 실적**: 주요 프로젝트 3-5개
5. **고객사 로고**: 신뢰 구축
6. **인증 및 수상**: 전문성 강조
7. **문의 CTA**: 명확한 행동 유도

#### 제품/솔루션 페이지
1. **제품 개요**: 핵심 특징 및 장점
2. **기술 사양**: 상세 스펙
3. **적용 사례**: 실제 현장 사진
4. **관련 자료**: 카탈로그 다운로드
5. **문의하기**: CTA

#### 시공 실적 페이지
1. **필터/검색**: 프로젝트 유형, 연도별
2. **프로젝트 카드**: 썸네일 + 기본 정보
3. **상세 정보**: 프로젝트명, 발주처, 기간, 규모
4. **현장 사진**: 갤러리 형태

## 4. 와이어프레임 요구사항

### 4.1 필수 제작 페이지
1. **메인 페이지** (데스크톱, 태블릿, 모바일)
2. **회사 소개** (데스크톱, 모바일)
3. **제품 목록 페이지** (데스크톱, 모바일)
4. **제품 상세 페이지** (데스크톱, 모바일)
5. **시공 실적 페이지** (데스크톱, 모바일)
6. **문의하기 페이지** (데스크톱, 모바일)

### 4.2 와이어프레임 상세도
- **Low-fidelity**: 정보 구조 및 레이아웃 확인용
- **Mid-fidelity**: 콘텐츠 배치 및 인터랙션 정의
- **주석 포함**: 각 요소의 기능 및 동작 설명

### 4.3 반응형 고려사항
- **데스크톱 (1200px+)**: 3-4 컬럼 레이아웃
- **태블릿 (768-1023px)**: 2 컬럼 레이아웃
- **모바일 (320-767px)**: 1 컬럼 레이아웃, 스택 구조

## 5. 인터랙션 및 기능 요구사항

### 5.1 네비게이션 인터랙션

#### 데스크톱
- **메인 메뉴 호버**: 서브 메뉴 드롭다운 (0.3초 딜레이)
- **Sticky Header**: 스크롤 시 상단 고정, 배경 불투명도 증가
- **현재 페이지 표시**: 메뉴 항목 하이라이트

#### 모바일
- **햄버거 메뉴**: 탭 시 전체 화면 오버레이
- **서브 메뉴**: 아코디언 방식 (+ / - 아이콘)
- **닫기**: X 버튼 또는 배경 탭

### 5.2 검색 기능
- **위치**: 헤더 우측 (돋보기 아이콘)
- **동작**: 클릭 시 검색창 확장
- **기능**: 
  - 실시간 자동완성 (제품명, 페이지명)
  - 최근 검색어 표시
  - 검색 결과 페이지 이동
- **기술**: JavaScript 기반 클라이언트 사이드 검색

### 5.3 이미지 갤러리
- **썸네일 그리드**: 클릭 시 라이트박스 팝업
- **라이트박스 기능**:
  - 이전/다음 이미지 네비게이션
  - 확대/축소
  - 닫기 (X 버튼 또는 ESC 키)
  - 이미지 설명 표시

### 5.4 슬라이더/캐러셀
- **자동 재생**: 5초 간격 (선택 사항)
- **수동 조작**: 좌우 화살표, 인디케이터 점
- **터치 제스처**: 모바일에서 스와이프
- **접근성**: 키보드 네비게이션 지원

### 5.5 문의 폼
- **필수 입력 필드**:
  - 이름 (텍스트)
  - 회사명 (텍스트)
  - 연락처 (전화번호)
  - 이메일 (이메일 형식 검증)
  - 문의 유형 (드롭다운: 제품 문의, 견적 요청, 기술 지원, 기타)
  - 문의 내용 (텍스트 영역)
- **선택 입력 필드**:
  - 관심 제품 (체크박스)
  - 파일 첨부 (PDF, 이미지)
- **유효성 검사**:
  - 실시간 검증 (포커스 아웃 시)
  - 에러 메시지 표시 (필드 하단, 빨간색)
  - 제출 전 전체 검증
- **제출 후**:
  - 성공 메시지 표시 또는 감사 페이지 이동
  - 이메일 전송 (관리자 + 사용자)

### 5.6 다운로드 기능
- **자료실**: PDF 파일 목록 (카탈로그, 매뉴얼, 기술 문서)
- **다운로드 버튼**: 명확한 아이콘 + 파일 크기 표시
- **다운로드 추적**: 클릭 이벤트 로깅 (Google Analytics)

### 5.7 지도 연동
- **오시는 길 페이지**: Google Maps 또는 Kakao Map 임베드
- **기능**:
  - 확대/축소
  - 길찾기 링크
  - 주소 복사 버튼

## 6. 콘텐츠 전략

### 6.1 B2B 콘텐츠 원칙
1. **신뢰 우선**: 실적, 인증, 고객사 전면 배치
2. **전문성 강조**: 기술 용어 사용하되 이해하기 쉽게 설명
3. **데이터 기반**: 구체적인 수치 및 사례 제시
4. **명확한 CTA**: 각 페이지마다 명확한 행동 유도

### 6.2 콘텐츠 톤앤매너
- **전문적이지만 친근하게**: 딱딱하지 않은 전문성
- **명확하고 간결하게**: 불필요한 수식어 제거
- **고객 중심**: "우리"보다 "고객님"의 관점
- **행동 지향**: 동사로 시작하는 문장

### 6.3 페이지별 핵심 메시지

#### 메인 페이지
- **히어로 메시지**: "현장의 모든 문제를 해결하는 솔루션 파트너"
- **서브 메시지**: "20년 경험, 1,000개 이상의 프로젝트 실적"
- **CTA**: "무료 상담 신청하기" / "제품 둘러보기"

#### 회사 소개
- **핵심 메시지**: "신뢰와 기술력으로 검증된 파트너"
- **강조 포인트**: 연혁, 인증, 수상, 주요 고객사

#### 제품/솔루션
- **핵심 메시지**: "현장 맞춤형 솔루션"
- **강조 포인트**: 기술 사양, 장점, 적용 사례

#### 시공 실적
- **핵심 메시지**: "검증된 실적, 성공적인 프로젝트"
- **강조 포인트**: 프로젝트 규모, 고객사, 현장 사진

### 6.4 이미지 전략
- **히어로 이미지**: 현장 작업 사진 (전문성 강조)
- **제품 이미지**: 고품질 제품 사진 (다각도)
- **실적 이미지**: 실제 현장 사진 (Before/After)
- **팀 이미지**: 전문가 팀 사진 (신뢰 구축)

### 6.5 비디오 콘텐츠 (선택 사항)
- **회사 소개 영상**: 30-60초 (메인 페이지)
- **제품 시연 영상**: 1-2분 (제품 페이지)
- **현장 인터뷰**: 고객 후기 (실적 페이지)

## 7. 접근성 요구사항

### 7.1 WCAG 2.1 AA 준수 사항

#### 인식 가능성 (Perceivable)
- **대체 텍스트**: 모든 이미지에 의미 있는 alt 속성
- **색상 대비**: 텍스트와 배경 대비 4.5:1 이상
- **텍스트 크기 조절**: 200%까지 확대 가능
- **비디오 자막**: 모든 비디오에 자막 제공

#### 운용 가능성 (Operable)
- **키보드 접근**: 모든 기능 키보드로 조작 가능
- **포커스 표시**: 명확한 포커스 인디케이터
- **충분한 시간**: 자동 슬라이더 일시정지 버튼
- **건너뛰기 링크**: 메인 콘텐츠로 바로 가기

#### 이해 가능성 (Understandable)
- **명확한 레이블**: 모든 폼 요소에 레이블 연결
- **에러 메시지**: 구체적이고 이해하기 쉬운 안내
- **일관된 네비게이션**: 모든 페이지에서 동일한 위치

#### 견고성 (Robust)
- **시맨틱 HTML**: 적절한 HTML 태그 사용
- **ARIA 속성**: 필요한 경우 ARIA 레이블 추가
- **스크린 리더 호환**: NVDA, JAWS 테스트

### 7.2 UX 디자이너 체크리스트
- [ ] 모든 인터랙티브 요소에 충분한 터치 영역 (최소 44x44px)
- [ ] 색상만으로 정보 전달하지 않기 (아이콘, 텍스트 병행)
- [ ] 명확한 포커스 순서 정의
- [ ] 에러 메시지 위치 및 스타일 정의
- [ ] 키보드 네비게이션 플로우 문서화

## 8. 반응형 디자인 가이드

### 8.1 브레이크포인트
```
Mobile: 320px - 767px
Tablet: 768px - 1023px
Desktop: 1024px - 1440px
Large Desktop: 1441px+
```

### 8.2 디바이스별 레이아웃 전략

#### 모바일 (320-767px)
- **레이아웃**: 1 컬럼, 세로 스택
- **네비게이션**: 햄버거 메뉴
- **이미지**: 전체 너비, 세로 비율 조정
- **폰트 크기**: 16px (본문), 24-32px (제목)
- **터치 영역**: 최소 44x44px
- **CTA 버튼**: 전체 너비 또는 최소 80% 너비

#### 태블릿 (768-1023px)
- **레이아웃**: 2 컬럼
- **네비게이션**: 햄버거 메뉴 또는 축약된 메뉴
- **이미지**: 2 컬럼 그리드
- **폰트 크기**: 16-18px (본문), 28-36px (제목)

#### 데스크톱 (1024px+)
- **레이아웃**: 3-4 컬럼
- **네비게이션**: 전체 메뉴 표시
- **이미지**: 3-4 컬럼 그리드
- **폰트 크기**: 18px (본문), 32-48px (제목)
- **최대 너비**: 1200px (콘텐츠 영역)

### 8.3 반응형 이미지
- **srcset 활용**: 디바이스별 최적 이미지 제공
- **WebP 형식**: 최신 브라우저 지원
- **Lazy Loading**: 스크롤 시 이미지 로드
- **비율 유지**: aspect-ratio 속성 활용

- **디바이스**: 데스크톱 (1920px), 모바일 (375px)

- 문의 폼 입력 및 검증
- 검색 기능

  3. 문의 폼 작성하기

  - 작업 완료 시간

  - 사용자 만족도 (5점 척도)
