# 시각적 확인 가이드

## 개요
이 문서는 플레이스홀더 이미지가 적용된 웹사이트를 브라우저에서 확인하는 방법을 안내합니다.

---

## 1. 로컬 서버 실행

### Python HTTP Server 사용
```bash
# 프로젝트 루트 디렉토리에서 실행
python -m http.server 8000
```

### 접속 URL
```
http://localhost:8000
```

## 3. 반응형 확인

### 3.1 데스크톱 (1920px)
```
Chrome DevTools > Toggle Device Toolbar (Ctrl+Shift+M)
Responsive > 1920 x 1080
```

**확인 사항:**
- [ ] 컨테이너 최대 너비 1280px
- [ ] 솔루션 카드 4열
- [ ] 고객사 로고 4열
- [ ] 프로젝트 카드 3열

### 3.2 태블릿 (768px)
```
Chrome DevTools > iPad
```

**확인 사항:**
- [ ] 솔루션 카드 2열
- [ ] 고객사 로고 4열
- [ ] 프로젝트 카드 2열
- [ ] 헤더 레이아웃 조정

### 3.3 모바일 (375px)
```
Chrome DevTools > iPhone SE
```

**확인 사항:**
- [ ] 햄버거 메뉴 표시
- [ ] 솔루션 카드 1열
- [ ] 통계 2열
- [ ] 고객사 로고 2-3열
- [ ] 프로젝트 카드 1열
- [ ] 푸터 1열

## 4. 인터랙션 테스트

### 4.1 모바일 메뉴
1. 햄버거 아이콘 클릭
2. 메뉴 열림 확인
3. 서브메뉴 토글 클릭
4. 서브메뉴 열림 확인
5. 메뉴 외부 클릭 또는 ESC 키
6. 메뉴 닫힘 확인

### 4.2 Hero 슬라이더
1. 자동 슬라이드 작동 확인 (5초 대기)
2. 좌측 화살표 클릭
3. 이전 슬라이드로 이동 확인
4. 우측 화살표 클릭
5. 다음 슬라이드로 이동 확인
6. 페이지네이션 점 클릭
7. 해당 슬라이드로 이동 확인

### 4.3 스크롤 효과
1. 페이지 하단으로 스크롤
2. Back to Top 버튼 표시 확인
3. Back to Top 버튼 클릭
4. 맨 위로 부드럽게 이동 확인
5. 통계 섹션으로 스크롤
6. 카운트업 애니메이션 작동 확인

### 4.4 호버 효과
1. 솔루션 카드에 마우스 오버
2. 카드 상승 효과 확인
3. 버튼에 마우스 오버
4. 버튼 색상 변경 확인
5. 링크에 마우스 오버
6. 링크 색상 변경 확인

## 9. 이슈 발견 시

### 9.1 이슈 기록
발견된 이슈는 다음 형식으로 기록:

```markdown
## 이슈 #1
- **카테고리**: 기능 / 디자인 / 성능 / 접근성
- **심각도**: 높음 / 중간 / 낮음
- **설명**: 이슈 상세 설명
- **재현 방법**: 
  1. 단계 1
  2. 단계 2
- **예상 결과**: 
- **실제 결과**: 
- **스크린샷**: (선택사항)
```

### 9.2 이슈 보고
- `docs/TEST-RESULTS.md`에 추가
- 또는 별도 이슈 트래커 사용
