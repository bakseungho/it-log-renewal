# 웹 기획 사용자 플로우

## 문서 개요

**프로젝트명**: (주)아이티로그 홈페이지 리뉴얼  
**작성일**: 2024  
**작성자**: Web Planner  
**버전**: 1.0

---

## 1. 사용자 플로우 개요

### 1.1 목적
- 주요 사용자 시나리오별 이동 경로 정의
- 전환 퍼널 최적화
- 이탈 방지 전략 수립
- UX 개선 포인트 식별

### 1.2 주요 사용자 목표
1. **정보 탐색**: 솔루션 정보 확인
2. **신뢰 확인**: 회사 실적 및 인증 확인
3. **문의/견적**: 상담 요청
4. **지원 요청**: 원격지원 또는 기술 지원

---

## 2. 주요 사용자 시나리오

### 2.1 시나리오 1: 신규 고객 - 솔루션 탐색 및 문의

**사용자 프로필:**
- 건설사 구매 담당자 (40대)
- 목표: 현장 안전 솔루션 도입 검토
- 디바이스: 데스크톱

**사용자 플로우:**

```
[검색 엔진] "건설 현장 안전 시스템"
    ↓
[메인 페이지 진입]
    ↓
[히어로 섹션 확인] - 핵심 메시지 파악
    ↓
[주요 솔루션 섹션 스크롤]
    ↓
[관심 솔루션 카드 클릭] "스마트 현장안전 시스템"
    ↓
[솔루션 상세 페이지]
    ↓
[시스템 개요 읽기]
    ↓
[시스템 구성도 확인]
    ↓
[주요 시공 사례 확인] - 신뢰도 검증
    ↓
[견적 문의 버튼 클릭]
    ↓
[문의하기 페이지]
    ↓
[문의 폼 작성]
    - 이름, 회사명, 연락처, 이메일, 내용 입력
    ↓
[개인정보 동의 체크]
    ↓
[문의하기 버튼 클릭]
    ↓
[제출 완료 메시지]
    ↓
[홈으로 가기 또는 페이지 종료]
```

**예상 소요 시간**: 5-8분  
**주요 터치포인트**: 메인 → 솔루션 상세 → 문의  
**전환 목표**: 문의 폼 제출

**이탈 방지 전략:**
- 솔루션 페이지에 명확한 CTA 배치
- 시공사례로 신뢰도 강화
- 문의 폼 간소화 (필수 항목 최소화)

---

### 2.2 시나리오 2: 기존 고객 - 원격지원 요청

**사용자 프로필:**
- 현장 관리자 (30대)
- 목표: 설치된 시스템 원격지원 요청
- 디바이스: 모바일

**사용자 플로우:**

```
[즐겨찾기 또는 직접 URL 입력]
    ↓
[메인 페이지 진입]
    ↓
[햄버거 메뉴 클릭]
    ↓
[고객지원 메뉴 클릭]
    ↓
[원격지원 클릭]
    ↓
[원격지원 페이지]
    ↓
[원격지원 서비스 연결 버튼 클릭]
    ↓
[외부 원격지원 사이트 이동] (새 창)
    ↓
[원격지원 진행]
```

**예상 소요 시간**: 1-2분  
**주요 터치포인트**: 메인 → 고객지원 → 원격지원  
**전환 목표**: 원격지원 사이트 접속

**이탈 방지 전략:**
- 메인 페이지에 원격지원 빠른 링크
- 모바일 최적화된 네비게이션
- 명확한 안내 문구

---

### 2.3 시나리오 3: 구매 담당자 - 회사 신뢰도 확인

**사용자 프로필:**
- 공공기관 구매 담당자 (50대)
- 목표: 업체 신뢰도 및 실적 확인
- 디바이스: 데스크톱

**사용자 플로우:**

```
[추천 또는 소개로 방문]
    ↓
[메인 페이지 진입]
    ↓
[회사소개 메뉴 호버]
    ↓
[회사소개 클릭]
    ↓
[회사소개 페이지]
    ↓
[기업정보 확인] - 설립연도, 규모 등
    ↓
[회사연혁 탭 클릭]
    ↓
[회사연혁 페이지]
    ↓
[타임라인 확인] - 주요 성과 검토
    ↓
[인증 및 특허 탭 클릭]
    ↓
[인증 및 특허 페이지]
    ↓
[인증서 이미지 확인] - ISO, 특허 등
    ↓
[메인 페이지로 돌아가기]
    ↓
[고객사 섹션 확인] - 주요 고객사 로고
    ↓
[주요 시공사례 확인]

[문의하기 버튼 클릭]
    ↓
[문의 폼 작성 및 제출]
```

**예상 소요 시간**: 8-12분  
**주요 터치포인트**: 회사소개 → 연혁 → 인증 → 고객사 → 문의  
**전환 목표**: 신뢰 구축 후 문의

**이탈 방지 전략:**
- 명확한 실적 데이터 제시
- 인증서 고해상도 이미지 제공
- 주요 고객사 로고 전면 배치

- 건설 현장 엔지니어 (35세)

```
    ↓
[검색 결과 표시]
    ↓
```

- 빠른 검색 기능

## 3. 전환 퍼널 분석

### 3.1 주요 전환 퍼널

```
[인지] 검색 엔진, 추천, 광고
    ↓ (100%)
[방문] 메인 페이지 진입
    ↓ (70%)
[탐색] 솔루션 또는 회사소개 확인
    ↓ (50%)
[평가] 시공사례, 인증, 고객사 확인
    ↓ (30%)
[관심] 문의 페이지 진입
    ↓ (20%)
[전환] 문의 폼 제출 또는 전화 문의
```

**목표 전환율**: 5% (방문자 대비 문의)

---

### 3.2 단계별 이탈 방지 전략

**인지 → 방문 (30% 이탈)**
- SEO 최적화로 검색 노출 향상
- 명확한 메타 설명으로 클릭 유도
- 빠른 페이지 로딩 속도

**방문 → 탐색 (30% 이탈)**
- 명확한 히어로 메시지
- 직관적인 네비게이션
- 주요 솔루션 전면 배치

**탐색 → 평가 (20% 이탈)**
- 상세한 솔루션 정보 제공
- 시공사례 이미지 및 설명
- 관련 솔루션 추천

**평가 → 관심 (20% 이탈)**
- 각 페이지에 명확한 CTA
- 신뢰 요소 강화 (인증, 고객사)
- 문의 프로세스 간소화

**관심 → 전환 (10% 이탈)**
- 간단한 문의 폼
- 실시간 입력 검증
- 다양한 문의 채널 (폼, 전화, 이메일)

## 4. 페이지 간 이동 경로

### 4.1 주요 이동 경로 맵

```
메인 페이지
├── 회사소개
│   ├── 회사소개
│   ├── CEO 인사말
│   ├── 회사연혁
│   ├── 인증 및 특허
│   └── 오시는 길
│
├── 솔루션 페이지 (6개)
│   ├── 스마트 현장안전 시스템
│   ├── 타워크레인 통합안전 시스템
│   ├── AI 영상 방송 관제시스템
│   ├── 스마트 환경센서 시스템
│   └── 스마트 출입통제 시스템
│
└── 고객지원
    ├── 원격지원
    └── 문의하기
```

---

### 4.2 크로스 링크 전략

**메인 페이지에서:**
- 주요 솔루션 카드 → 각 솔루션 상세 페이지
- 시공사례 카드 → 해당 솔루션 페이지의 사례 섹션
- 고객사 섹션 → 회사소개 페이지
- 문의 CTA → 문의하기 페이지

**솔루션 페이지에서:**
- 관련 솔루션 → 다른 솔루션 페이지
- 문의 CTA → 문의하기 페이지
- 브레드크럼 → 상위 페이지

**회사소개 페이지에서:**
- 주요 솔루션 언급 → 솔루션 페이지
- 문의 CTA → 문의하기 페이지

## 5. 사용자 여정 맵

### 5.1 신규 고객 여정

| 단계 | 사용자 행동 | 사용자 생각 | 감정 | 터치포인트 | 페인포인트 | 개선 기회 |
|------|------------|------------|------|-----------|-----------|----------|
| **인지** | 검색 엔진에서 검색 | "현장 안전 솔루션이 필요해" | 중립 | 검색 결과 | 정보 부족 | SEO 최적화 |
| **방문** | 메인 페이지 진입 | "이 회사가 뭘 하는 곳이지?" | 호기심 | 히어로 섹션 | 메시지 불명확 | 명확한 가치 제안 |
| **탐색** | 솔루션 페이지 확인 | "우리 현장에 맞을까?" | 관심 | 솔루션 상세 | 정보 과다/부족 | 적절한 정보량 |
| **평가** | 시공사례 확인 | "실제로 잘 작동할까?" | 의구심 | 시공사례 | 사례 부족 | 다양한 사례 제공 |
| **결정** | 문의 폼 작성 | "문의해볼까?" | 긍정적 | 문의 페이지 | 폼 복잡 | 간소화된 폼 |
| **전환** | 문의 제출 | "빨리 연락 오면 좋겠다" | 기대 | 완료 메시지 | 응답 시간 불명확 | 응답 시간 안내 |

### 5.2 기존 고객 여정

| 단계 | 사용자 행동 | 사용자 생각 | 감정 | 터치포인트 | 페인포인트 | 개선 기회 |
|------|------------|------------|------|-----------|-----------|----------|
| **재방문** | 즐겨찾기로 접속 | "원격지원 받아야지" | 급함 | 메인 페이지 | 원격지원 찾기 어려움 | 빠른 링크 제공 |
| **탐색** | 고객지원 메뉴 클릭 | "어디 있지?" | 조급함 | 네비게이션 | 메뉴 구조 복잡 | 명확한 메뉴명 |
| **실행** | 원격지원 연결 | "빨리 연결되면 좋겠다" | 불안 | 원격지원 페이지 | 연결 지연 | 빠른 연결 |
| **완료** | 지원 받음 | "문제 해결됐다" | 만족 | 외부 사이트 | - | - |

## 6. 모바일 사용자 플로우

- 터치 인터랙션 최적화
- 클릭투콜 활용
- 간소화된 폼

**주요 플로우:**


## 7. 이탈 시나리오 및 대응

### 7.1 주요 이탈 시나리오

**시나리오 1: 메인 페이지에서 즉시 이탈**
- 원인: 메시지 불명확, 로딩 느림
- 대응: 명확한 히어로 메시지, 성능 최적화

**시나리오 2: 솔루션 페이지에서 이탈**
- 원인: 정보 부족, 신뢰도 낮음
- 대응: 상세 정보 제공, 시공사례 강화

**시나리오 3: 문의 페이지에서 이탈**
- 원인: 폼 복잡, 개인정보 우려
- 대응: 폼 간소화, 개인정보 보호 명시

---

### 7.2 재참여 전략

**이탈 후 재방문 유도:**
- 명확한 브랜드 메시지로 기억에 남기
- 즐겨찾기 유도 (모바일 홈 화면 추가)
- 뉴스레터 구독 (선택)
