# 홈페이지 리뉴얼 프로젝트 - 작업 배분 및 일정

## 문서 개요

**프로젝트명**: (주)아이티로그 홈페이지 리뉴얼 프로젝트  
**작성일**: 2024  
**작성자**: PM (Product Manager)  
**버전**: 1.0

---

## 1. 프로젝트 개요

### 1.1 프로젝트 목표
- 건설 현장 솔루션 전문 기업의 브랜드 이미지 강화
- 사용자 중심의 직관적인 정보 구조 구축
- B2B 고객의 의사결정을 지원하는 신뢰성 있는 콘텐츠 제공
- 모바일 최적화 및 접근성 향상
- 문의 전환율 30% 증가 목표

### 1.2 프로젝트 범위
- **사이트 유형**: 정적 웹사이트 (HTML/CSS/JavaScript)
- **주요 페이지**: 메인, 회사소개, 솔루션(3개 카테고리), 고객지원
- **핵심 기능**: 문의하기, 원격지원 연결, 지도 연동
- **제외 기능**: 로그인, 회원가입, 게시판, 실시간 채팅

### 1.3 타겟 사용자
- **1차 타겟**: 건설사/시공사 구매 의사결정자 (40-50대)
- **2차 타겟**: 현장 담당자 및 엔지니어 (30-40대)
- **3차 타겟**: 공공기관 구매 담당자 (30-40대)

## 4. 세부 작업 배분

### 4.1 웹 기획자 (web-planner-website)

#### 담당 업무
1. **정보 구조 설계 (Week 1)**
   - 사이트맵 작성
   - 정보 구조도(IA) 설계
   - 네비게이션 구조 정의
   - 콘텐츠 계층 구조 설계

2. **콘텐츠 전략 수립 (Week 1)**
   - 페이지별 콘텐츠 요구사항 정의
   - 콘텐츠 우선순위 설정
   - SEO 키워드 전략
   - 콘텐츠 톤앤매너 가이드

3. **기능 명세 작성 (Week 2)**
   - 페이지별 기능 요구사항
   - 인터랙션 정의
   - 외부 서비스 연동 명세 (지도, 문의 폼)
   - 에러 처리 및 예외 상황 정의

4. **사용자 플로우 설계 (Week 2)**
   - 주요 사용자 시나리오 정의
   - 사용자 여정 맵 작성
   - 전환 퍼널 설계
   - 페이지 간 이동 경로 최적화

#### 산출물
- `docs/web-planning-sitemap.md` - 사이트맵 및 IA
- `docs/web-planning-content-strategy.md` - 콘텐츠 전략
- `docs/web-planning-functional-spec.md` - 기능 명세서
- `docs/web-planning-user-flow.md` - 사용자 플로우

#### 작업 지시서
- `docs/pm-brief-for-web-planner.md` 참조

### 4.2 UX 디자이너 (ux-designer-website)

#### 담당 업무
1. **사용자 리서치 (Week 3 초)**
   - 페르소나 정의
   - 사용자 니즈 분석
   - 경쟁사 벤치마킹

2. **와이어프레임 제작 (Week 3)**
   - 메인 페이지 (데스크톱/모바일)
   - 회사소개 페이지

   - 솔루션 상세 페이지
   - 고객지원 페이지

3. **프로토타입 제작 (Week 4)**
   - 인터랙티브 프로토타입 (Figma)
   - 주요 사용자 플로우 구현
   - 인터랙션 정의

4. **디자인 시스템 구축 (Week 4)**
   - 컬러 팔레트
   - 타이포그래피 시스템
   - UI 컴포넌트 라이브러리
   - 간격 및 그리드 시스템

### 4.3 프론트엔드 개발자 (frontend-dev-website)

#### 담당 업무
1. **프로젝트 셋업 (Week 5 초)**
   - 디렉토리 구조 설정
   - 개발 환경 구축
   - CSS 변수 및 공통 스타일 작성

2. **HTML/CSS 마크업 (Week 5-6)**
   - 시맨틱 HTML 구조
   - 반응형 CSS (Mobile First)
   - 접근성 준수 (WCAG 2.1 AA)
   - 크로스 브라우저 호환성

3. **JavaScript 기능 구현 (Week 6-7)**
   - 네비게이션 및 모바일 메뉴
   - 슬라이더/캐러셀 (Swiper.js)
   - 문의 폼 검증 및 제출
   - 검색 기능
   - 이미지 갤러리
   - 지도 연동

4. **성능 최적화 (Week 7)**
   - 이미지 최적화 (WebP, Lazy Loading)
   - 코드 압축 및 번들링
   - 캐싱 전략
   - Lighthouse 점수 90점 이상 달성

5. **테스트 및 배포 (Week 7-8)**
   - 기능 테스트
   - 반응형 테스트
   - 브라우저 호환성 테스트
   - 접근성 테스트
   - 배포 및 모니터링

#### 산출물
- 완성된 정적 웹사이트 (HTML/CSS/JS)
- `docs/frontend-technical-documentation.md` - 기술 문서
- `docs/frontend-deployment-guide.md` - 배포 가이드

#### 작업 지시서
- `docs/pm-brief-for-frontend-dev.md` 참조 (업데이트 예정)

## 5. 의존성 및 작업 순서

### 5.1 작업 의존성 다이어그램
```
PM 요구사항 정의
    ↓
웹 기획자: IA 설계 + 콘텐츠 전략
    ↓
웹 기획자: 기능 명세 + 사용자 플로우
    ↓
UX 디자이너: 와이어프레임
    ↓
UX 디자이너: 프로토타입 + 디자인 시스템
    ↓
프론트엔드 개발자: HTML/CSS 마크업
    ↓
프론트엔드 개발자: JavaScript 기능 구현
    ↓
전체 팀: 테스트 및 배포
```

### 5.2 병렬 작업 가능 구간
- Week 3: 웹 기획자의 기능 명세 작성과 UX 디자이너의 리서치는 병렬 진행 가능
- Week 6: 프론트엔드 개발자의 일부 페이지 개발과 다른 페이지 마크업은 병렬 진행 가능

### 5.3 크리티컬 패스
```
IA 설계 → 와이어프레임 → 프로토타입 → HTML/CSS 마크업 → JavaScript 구현 → 배포
```

## 9. 품질 기준

### 9.1 기획 품질 기준
- [ ] 사이트맵이 비즈니스 목표와 일치
- [ ] 모든 페이지의 콘텐츠 요구사항 명확히 정의
- [ ] 사용자 플로우가 논리적이고 효율적
- [ ] 기능 명세가 구체적이고 실행 가능

### 9.2 UX 품질 기준
- [ ] 와이어프레임이 정보 구조를 정확히 반영
- [ ] 프로토타입이 주요 사용자 플로우 구현
- [ ] 디자인 시스템이 일관성 있고 확장 가능
- [ ] 접근성 가이드라인 준수 (WCAG 2.1 AA)

### 9.3 개발 품질 기준
- [ ] 시맨틱 HTML 사용
- [ ] 반응형 디자인 완벽 구현 (320px ~ 1920px)
- [ ] 크로스 브라우저 호환성 (Chrome, Firefox, Safari, Edge)
- [ ] Lighthouse 점수 90점 이상 (모든 항목)
- [ ] 접근성 테스트 통과 (WAVE 에러 0개)
- [ ] 페이지 로딩 속도 3초 이내
