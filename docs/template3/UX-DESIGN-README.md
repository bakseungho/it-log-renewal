## 🎯 프로젝트 개요

### 디자인 컨셉
**"Dark Elegance & Tech Premium"**

Template3는 다크 모드 기반의 프리미엄 웹사이트로, 건설 안전 솔루션 업계에서 차별화된 고급 브랜드 이미지를 구축합니다. https://www.dwce.co.kr/ 사이트를 참고하여 풀스크린 레이아웃, 대형 타이포그래피, 스크롤 기반 인터랙션을 적용했습니다.

### 핵심 특징
- ✨ **다크 모드 기반**: #0a0a0a 배경의 고급스러운 디자인
- 🎨 **듀얼 포인트 컬러**: 골드(#d4af37) + 시안(#00d9ff)
- 📱 **풀스크린 경험**: 100vh 히어로 섹션과 몰입형 레이아웃
- 🎬 **GSAP 애니메이션**: 스크롤 기반 프리미엄 인터랙션
- 🏆 **프리미엄 브랜드**: 대형 프로젝트 타겟 고급 이미지

## 🎯 핵심 디자인 원칙

### 1. 다크 모드 가독성 최우선
- 텍스트 대비 4.5:1 이상
- 화이트 텍스트 기본
- 충분한 행간 (1.6-1.8)

### 2. 계층적 배경 사용
- Level 1: #0a0a0a (메인)
- Level 2: #1a1a1a (카드)
- Level 3: #2a2a2a (호버)

### 3. 포인트 컬러 절제
- 골드: 프리미엄 요소만
- 시안: 기술 관련만
- 과도한 사용 지양

### 4. 넉넉한 여백
- 섹션 간: 최소 80px
- 요소 간: 최소 40px
- 텍스트: 최소 24px

### 5. 대형 비주얼
- 히어로: 1920x1080 이상
- 고해상도 유지
- Retina 대응

### 6. 임팩트 타이포그래피
- 제목: 60px 이상
- 굵은 웨이트 (Bold, Black)
- 명확한 메시지

### 7. 부드러운 애니메이션
- 지속 시간: 0.3-1.2초
- Ease: power2.out
- 60fps 유지

### 8. WCAG 2.1 AA 준수
- 색상 대비 확보
- 키보드 네비게이션
- 스크린 리더 호환

### 9. 명확한 포커스 표시
- 골드 아웃라인 2px
- 모든 인터랙티브 요소
- 키보드 탭 순서 논리적

### 10. 성능 최적화
- 2초 이내 로딩
- 60fps 애니메이션
- 이미지 최적화

## 🔧 개발 가이드

### CSS 변수 사용

```css
/* 컬러 */
background: var(--color-bg-primary);
color: var(--color-text-primary);
border-color: var(--color-accent-gold);

/* 타이포그래피 */
font-size: var(--font-size-h1);
font-family: var(--font-primary);

/* 간격 */
padding: var(--spacing-xl);
margin-bottom: var(--spacing-md);

/* 애니메이션 */
transition: all var(--transition-base) var(--ease-smooth);
```

### GSAP 애니메이션

```javascript
// 페이드인
gsap.from(".fade-in", {
  opacity: 0,
  y: 50,
  duration: 1,
  scrollTrigger: {
    trigger: ".fade-in",
    start: "top 80%"
  }
});

// 카운트업
gsap.to(".stat-number", {
  innerText: 1500,
  duration: 2,
  snap: { innerText: 1 },
  scrollTrigger: {
    trigger: ".stat-number",
    start: "top 80%"
  }
});
```

### 반응형 미디어 쿼리

```css
/* 모바일 기본 */
.hero-title {

/* 태블릿 */

  }
}

/* 데스크톱 */
```

## ❓ FAQ

### Q1. 다크 모드에서 가독성이 떨어지지 않나요?
**A**: 텍스트 대비를 4.5:1 이상으로 유지하고, 화이트 텍스트와 충분한 행간(1.6-1.8)을 사용하여 가독성을 확보했습니다. WCAG 2.1 AA 기준을 준수합니다.

### Q2. 골드와 시안 두 가지 포인트 컬러를 사용하는 이유는?
**A**: 골드는 프리미엄과 신뢰를, 시안은 기술력과 혁신을 상징합니다. 각각의 의미를 명확히 구분하여 사용하면 브랜드 메시지가 더 효과적으로 전달됩니다.

### Q3. 풀스크린 히어로 섹션이 모바일에서도 필요한가요?
**A**: 모바일에서는 70vh로 조정하여 스크롤 유도와 콘텐츠 접근성의 균형을 맞췄습니다.

### Q4. GSAP 애니메이션이 성능에 영향을 주지 않나요?
**A**: GSAP는 GPU 가속을 활용하여 60fps를 유지합니다. 모바일에서는 애니메이션을 간소화하고 패럴랙스를 비활성화하여 성능을 최적화했습니다.

### Q5. Template1, Template2와의 차이점은?
**A**: 
- Template1: 클래식 비즈니스 (밝은 배경, 블루)
- Template2: 모던 비즈니스 (화이트 배경, 그린)
- Template3: 다크 프리미엄 (다크 배경, 골드+시안)

Template3는 프리미엄 고객층을 타겟으로 한 차별화된 디자인입니다.

### Q6. 접근성 테스트는 어떻게 하나요?
**A**: 
- Lighthouse 접근성 점수 확인
- 키보드만으로 전체 네비게이션 테스트
- 스크린 리더 (NVDA, JAWS) 테스트
- 색상 대비 도구 (WebAIM Contrast Checker) 사용

### Q7. 디자인 시스템을 수정하고 싶은데 어떻게 하나요?
**A**: CSS 변수를 수정하면 전체 사이트에 일괄 적용됩니다. ux-design-system.md의 CSS 변수 섹션을 참고하세요.
